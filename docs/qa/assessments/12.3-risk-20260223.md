## Risk Details

### TECH-001: Empty model parameter

**Score: 1 (Minimal)**
**Location:** `scripts/local_rag.py:344, 441`
**Description:** The `model=""` parameter is passed to `run_agentic_loop()`. The agentic module handles this by letting the API determine the model.
**Mitigation:** This is acceptable - the Ollama/LM Studio API uses the loaded model. Could be enhanced to detect model from API response.
**Testing Focus:** Integration tests with real API.

### UX-001: Different REPL prompts

**Score: 1 (Minimal)**
**Location:** `scripts/local_rag.py:317`
**Description:** The agentic mode uses `[Agentic] You:` prefix while simple RAG uses `You:`. This clearly indicates mode but changes user experience.
**Mitigation:** This is intentional per AC4/AC7 to clearly indicate which mode is active.
**Testing Focus:** Tests verify prompt formats (`TestREPLPrompt`).

### REL-001: Import dependency at module level

**Score: 2 (Low)**
**Location:** `scripts/local_rag.py:50-51`
**Description:** Agentic and search_corpus_tool imports are at module level. If these fail, the entire CLI fails even in simple RAG mode.
**Mitigation:** Could be changed to lazy imports, but current approach is cleaner and imports are fast.
**Testing Focus:** Existing tests verify imports work.
