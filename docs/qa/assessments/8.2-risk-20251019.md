## Risk Details

### PERF-001: Formula Detection Accuracy

**Category:** Performance
**Score:** 4 (Medium)
**Probability:** Medium - Heuristic-based detection may not catch all formula variations
**Impact:** Medium - Missed formulas reduce extraction completeness

**Description:**
The `detect_formula_regions()` function uses connected component analysis with hardcoded heuristics:
- Minimum area threshold (200 pixels)
- Aspect ratio limits (0.5 to 20)
- Centered formula detection (20% tolerance)

These may not work well for:
- Formulas with unusual layouts
- Dense mathematical pages
- Formulas in tables or complex layouts
- PDFs with varying resolutions

**Affected Components:**
- `formula_extractor.py:93-157` - Detection logic
- `formula_extractor.py:138-145` - Hardcoded thresholds

**Detection Method:** Code review and story requirements analysis

**Mitigation Strategy:**

**Type:** Preventive + Detective

**Actions:**
1. Add integration tests with real scientific PDFs to measure actual recall
2. Expose thresholds as configuration parameters for tuning
3. Add logging of detection statistics (formulas detected, pages scanned)
4. Consider ML-based formula detection in future iteration
5. Document known limitations in CLAUDE.md

**Testing Requirements:**
- Integration test with known ground truth (AC4 requires >75% recall)
- Test various PDF types (equations, scientific papers, mixed content)
- Measure false positive and false negative rates

**Residual Risk:** Low-Medium - Some formulas will be missed, but error recovery prevents pipeline failures

**Owner:** dev
**Timeline:** Future enhancement (post-MVP)

### DATA-001: pix2tex LaTeX Output Quality

**Category:** Data Quality
**Score:** 4 (Medium)
**Probability:** Medium - pix2tex has 80-85% accuracy per Story 8.1 research
**Impact:** Medium - Incorrect LaTeX produces unusable formulas

**Description:**
The pix2tex model may produce:
- Incorrect LaTeX syntax
- Misrecognized symbols
- Missing terms or operators
- Hallucinated content

Story 8.1 research showed 80-85% accuracy, meaning 15-20% of formulas may have errors.

**Affected Components:**
- `formula_extractor.py:240-247` - pix2tex extraction
- No validation of LaTeX output currently implemented

**Detection Method:** Story 8.1 research findings

**Mitigation Strategy:**

**Type:** Detective + Corrective

**Actions:**
1. Log all extracted LaTeX for manual review during early adoption
2. Add LaTeX syntax validation in Story 8.3 (formula formatter)
3. Consider confidence scoring if pix2tex provides it
4. Implement LaTeX compilation test to detect syntax errors
5. Track LaTeX error rates in production

**Testing Requirements:**
- Unit test validates empty LaTeX is skipped (formula_extractor.py:250-252)
- Future: Add LaTeX syntax validation tests
- Future: Compare extracted LaTeX against known ground truth

**Residual Risk:** Medium - Some incorrect LaTeX will reach output, but per-formula error handling prevents cascading failures

**Owner:** dev
**Timeline:** LaTeX validation in Story 8.3
