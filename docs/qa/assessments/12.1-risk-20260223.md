## Risk Details

### TECH-001: Non-unique tool_call_id

**Score: 1 (Minimal)**
**Location:** `scripts/agentic.py:318`
**Description:** The `tool_call_id` is formatted as `{tc.name}_{iteration}` which may not be unique if the same tool is called multiple times in one turn.
**Mitigation:** Future enhancement could add a sequence counter per turn.
**Testing Focus:** Current tests use single tool per call; edge case with same tool called twice in one turn.

### PERF-001: No retry logic

**Score: 4 (Medium)**
**Location:** `scripts/agentic.py:260-275`
**Description:** API requests fail immediately on transient network errors without retry.
**Mitigation:** Consider adding exponential backoff for production use. Currently acceptable for local LLM use case where network failures are rare.
**Testing Focus:** Integration tests with real Ollama instance.

### SEC-001: No URL validation

**Score: 1 (Minimal)**
**Location:** `scripts/agentic.py:125-169`
**Description:** The `api_url` parameter is not validated for format or scheme.
**Mitigation:** Low risk as this is a local tool with developer-provided URLs.
**Testing Focus:** N/A - caller responsibility.

### DATA-001: JSON parse fallback

**Score: 1 (Minimal)**
**Location:** `scripts/agentic.py:192-196`
**Description:** When tool arguments JSON parsing fails, the raw string is stored in a `{"raw": ...}` dict rather than failing.
**Mitigation:** This is actually graceful degradation, allowing the tool to potentially handle the raw string.
**Testing Focus:** Test `test_parse_tool_call_args_as_string` covers JSON string parsing.
