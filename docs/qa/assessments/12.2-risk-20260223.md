## Risk Details

### PERF-001: No embedder caching

**Score: 2 (Low)**
**Location:** `scripts/search_corpus_tool.py:174`
**Description:** The embedder is passed in as a parameter and its lifecycle is managed externally. However, if instantiated per-call, the model loading could be slow.
**Mitigation:** Acceptable - the factory pattern expects the embedder to be pre-loaded and reused.
**Testing Focus:** Verify embedder is called correctly (test exists).

### DATA-001: Silent YAML fallback

**Score: 1 (Minimal)**
**Location:** `scripts/search_corpus_tool.py:239-244`
**Description:** If YAML front matter parsing fails, the exception is caught and the full content is used as body. This is graceful but could mask corrupted metadata.
**Mitigation:** Logging could be added for debugging, but current behavior is appropriate for user-facing tool.
**Testing Focus:** Test handles chunks with and without YAML front matter.

### TECH-001: Positional chunk_map indexing

**Score: 1 (Minimal)**
**Location:** `scripts/search_corpus_tool.py:182-185`
**Description:** Chunks are accessed by FAISS index position, not by a stable chunk_id. If chunk_map order changes relative to FAISS index, results would be incorrect.
**Mitigation:** This matches the existing local_rag.py pattern - the chunk_map and FAISS index are generated together and kept in sync.
**Testing Focus:** Tests verify correct indexing behavior.
