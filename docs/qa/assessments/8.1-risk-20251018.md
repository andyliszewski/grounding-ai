## Detailed Risk Register

### Medium Risks

#### TECH-001: pix2tex Accuracy Variance

**Score: 4 (Medium Risk)**
**Probability**: Medium (2) - Formula extraction accuracy may vary with complex notation
**Impact**: Medium (2) - Could require fallback to Nougat or alternative approach

**Description**: While pix2tex claims BLEU 0.88 accuracy, real-world performance may vary depending on:
- Formula complexity (matrices, nested fractions, specialized symbols)
- Image quality (resolution, scan artifacts, handwriting vs. print)
- Mathematical notation styles (different LaTeX rendering approaches)

**Affected Components**:
- pix2tex tool selection
- Epic 8 implementation success
- Story 8.2 integration approach

**Detection Method**: Documented in comparison matrix and installation guide

**Mitigation Strategy**:

**Type**: Preventive + Corrective

**Actions**:
1. ✅ **Completed**: Document fallback options (Nougat, RapidLaTeXOCR) in comparison matrix
2. ✅ **Completed**: Set realistic accuracy expectations (80-85%) in Epic 8 planning
3. **Story 8.2**: Implement error handling for extraction failures
4. **Story 8.3**: Add human-in-the-loop review option for critical formulas
5. **Story 8.5**: Include accuracy testing in integration test suite

**Testing Requirements**:
- PoC tested simple formula (✅ Completed)
- Story 8.2: Test with diverse formula types (simple → complex)
- Story 8.5: Create regression test corpus with known-good formulas

**Residual Risk**: Low - Fallback options available, expectations set appropriately

**Owner**: Dev (for Stories 8.2-8.5)
**Timeline**: Ongoing through Epic 8 implementation

### Low Risks

#### PERF-001: Model Initialization Overhead

**Score: 2 (Low Risk)**
**Probability**: Low (1) - Known issue with documented mitigation
**Impact**: Medium (2) - ~19s cold start could impact user experience for single-formula extraction

**Description**: pix2tex model requires ~19s initialization on first use (model download + loading). This creates a noticeable delay for:
- First formula extraction in a session
- CLI invocations without model caching

**Affected Components**:
- pdf2llm CLI responsiveness
- User experience for formula extraction feature

**Detection Method**: Measured in PoC (poc_pix2tex.py)

**Mitigation Strategy**:

**Type**: Preventive

**Actions**:
1. ✅ **Completed**: Document performance characteristics in installation guide
2. ✅ **Completed**: Include caching pattern in comparison matrix
3. **Story 8.2**: Implement model instance caching for batch processing
4. **Story 8.5**: Add progress indicator for first-time model download
5. **Future Optimization**: Consider RapidLaTeXOCR (ONNX) for faster cold starts

**Testing Requirements**:
- Batch processing performance test (Story 8.5)
- CLI responsiveness test with cached vs. uncached model

**Residual Risk**: Very Low - Well-documented with clear mitigation path

**Owner**: Dev (Story 8.2 for caching implementation)
**Timeline**: Story 8.2 (model caching)

**Risk**: pix2tex (BLEU 0.88) has lower accuracy than Nougat (90%+)

**Rationale for Acceptance**:
- Project prioritizes ease of integration over maximum accuracy
- pix2tex is production-ready vs. Nougat research-grade
- pix2tex MIT license vs. Nougat CC-BY-NC (commercial blocker)
- pix2tex is better for isolated formulas vs. Nougat full-page approach
- 80-85% accuracy is sufficient for Epic 8 MVP goals

**Sign-off**: Documented in comparison matrix rationale
**Mitigation**: Fallback to Nougat available if accuracy proves insufficient
