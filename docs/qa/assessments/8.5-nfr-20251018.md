### Performance: CONCERNS

**Evidence:**
- Model initialization in controller.py happens per-file (not cached globally)
- pix2tex model loading adds ~1-2s overhead per PDF
- Processing overhead documented as ~10-15% slower than text-only

**Issues:**
1. **Model re-initialization overhead** (Performance)
   - Risk: Unnecessary latency in batch processing
   - Impact: ~1-2s per PDF when processing multiple files
   - Fix: Implement lazy singleton pattern for pix2tex model instance

**Recommendation:**
```python
# In formula_extractor.py - implement module-level caching
_model_instance = None

def get_model():
    global _model_instance
    if _model_instance is None:
        from pix2tex.cli import LatexOCR
        _model_instance = LatexOCR()
    return _model_instance
```

**Target:** <2s initialization overhead across entire batch run (not per-file)
