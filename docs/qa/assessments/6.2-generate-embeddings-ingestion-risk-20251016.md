## Risk Details

### PERF-001: Slow Embedding Generation

**Score: 2 (Low)**
**Probability**: Low - Model is optimized, hardware typically sufficient
**Impact**: Medium - Could slow down large batch processing
**Affected Components**: controller.py:92-144

**Mitigation**:
- Warning logged if >1s per chunk detected
- Performance metrics tracked for monitoring
- Can be disabled via flag if problematic

**Residual Risk**: Very low - well-monitored and opt-in

### PERF-002: Memory Growth with Large Corpus

**Score: 2 (Low)**
**Probability**: Low - Only for very large corpora (>10k chunks)
**Impact**: Medium - Could cause memory pressure
**Affected Components**: pipeline.py:43 (embeddings dict)

**Mitigation**:
- Memory overhead calculated: ~1.5KB per embedding
- 10,000 chunks = ~15MB (acceptable)
- Dict cleared after vector store write (Story 6.3)

**Residual Risk**: Low - manageable within typical use cases

### TECH-001: Model Download Failure on First Run

**Score: 4 (Medium)**
**Probability**: Medium - Network issues, disk space
**Impact**: Medium - First run fails, user confusion
**Affected Components**: embedder.py model loading

**Mitigation**:
- Model caching after first successful download
- Clear error messages from embedder module
- Story 6.1 already handles this gracefully

**Testing Requirements**:
- Test with clean cache (model not downloaded)
- Verify error message clarity

**Residual Risk**: Low - well-handled by existing code

### OPS-001: Increased Pipeline Runtime Affects Workflows

**Score: 4 (Medium)**
**Probability**: Medium - Embeddings add processing time
**Impact**: Medium - User workflows may need adjustment
**Affected Components**: Overall pipeline

**Mitigation**:
- Flag defaults to False (opt-in)
- Progress bar shows "(embeddings)" indicator
- Performance warnings guide users
- Can disable if too slow

**Testing Requirements**:
- Measure runtime delta with/without embeddings
- Verify progress reporting works

**Residual Risk**: Low - user controls via flag
