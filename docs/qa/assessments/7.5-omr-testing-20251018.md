**Timeout Analysis**:
- Current timeout: 120 seconds (defined in omr_parser.py)
- Large orchestral scores (Maria - 3.4MB) exceed this limit
- Recommendation: Consider configurable timeout via CLI flag for production use

## Warnings Observed

**Warning**: "Missing doc_hashes for slug=..." (5 occurrences)

**Analysis**:
- This warning is expected for OMR pipeline
- OMR processing doesn't compute document hashes the same way as text pipeline
- Documents are hashed from PDF bytes (file SHA-1 for doc_id)
- Not a blocker: Manifest correctly populated with doc_id from file hash

**Recommendation**: Update warning message to clarify this is expected for OMR mode, or suppress for `--parser omr`.

## Known Limitations Discovered

1. **Timeout Threshold**: 120 seconds insufficient for large orchestral scores
2. **Complex PDFs**: West Side Story (3.4MB, likely 50+ pages) exceeded processing capacity
3. **No Progress Indication**: Audiveris subprocess provides no progress updates during long processing
