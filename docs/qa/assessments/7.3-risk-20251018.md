## Identified Risks

### RISK-001: Manual ABC Generation Implementation
**Severity:** MEDIUM
**Likelihood:** MEDIUM
**Impact:** MEDIUM
**Category:** Technical
**Status:** MITIGATED

**Description:**
music21's ABC export failed with "no output extensions registered" error, requiring custom ABC notation generator. Manual implementation increases complexity and potential for ABC spec compliance issues.

**Evidence:**
```python
# music_formatter.py:176-256
# Manual ABC generation instead of music21 export
abc_content = "X:1\n"
abc_content += "T:Untitled\n"
# ... manual pitch/octave/duration conversion
```

**Impact Analysis:**
- Custom ABC generator may not handle all ABC notation features (repeats, ties, ornaments)
- Potential for ABC syntax errors not caught by automated validation
- Maintenance burden if ABC spec changes
- Limited to basic ABC features currently implemented

**Mitigation Implemented:**
1. **Test Coverage:** 3 dedicated ABC tests verify basic functionality
2. **Header Generation:** Proper X, T, M, L, K fields
3. **Pitch Conversion:** Handles octaves (C4=C, C5=c, C3=C,) and accidentals (^F for F#)
4. **Barlines:** Measures separated by |
5. **Duration Support:** Quarter, eighth (/2), half (2) notes

**Current Limitations:**
- No external validation with abc2midi (AC3 originally specified)
- Limited duration support (only 0.25, 0.5, 1.0, 2.0)
- No support for: ties, repeats, ornaments, grace notes, chords

**Recommendation:** ACCEPT with caveats - Basic ABC generation works for simple melodies. Consider adding abc2midi validation in future story if advanced ABC features needed.

### RISK-004: Temp File Cleanup for MIDI
**Severity:** LOW
**Likelihood:** LOW
**Impact:** LOW
**Category:** Resource Management
**Status:** MITIGATED

**Description:**
format_to_midi() creates temporary files via tempfile.NamedTemporaryFile and manually deletes them. Potential for temp file leaks if exception occurs between file creation and deletion.

**Evidence:**
```python
# music_formatter.py:288-295
with tempfile.NamedTemporaryFile(suffix='.mid', delete=False) as tmp:
    tmp_path = tmp.name

s.write('midi', fp=tmp_path)

# Read the MIDI file bytes
with open(tmp_path, 'rb') as f:
    midi_bytes = f.read()

# Clean up temp file
Path(tmp_path).unlink()
```

**Impact Analysis:**
- If exception occurs after s.write() but before unlink(), temp file remains
- Potential /tmp directory pollution over time
- Small disk space impact (MIDI files typically <1MB)

**Mitigation Implemented:**
1. **Try-Except Wrapping:** Entire function wrapped in try-except
2. **FormattingError:** Exceptions raised, allowing cleanup in higher-level error handler
3. **Tempfile Module:** Uses tempfile.NamedTemporaryFile for proper naming

**Improvement Suggestion:**
```python
# Better approach with try-finally
tmp_path = None
try:
    with tempfile.NamedTemporaryFile(suffix='.mid', delete=False) as tmp:
        tmp_path = tmp.name
    s.write('midi', fp=tmp_path)
    with open(tmp_path, 'rb') as f:
        midi_bytes = f.read()
    return midi_bytes
finally:
    if tmp_path and Path(tmp_path).exists():
        Path(tmp_path).unlink()
```

**Recommendation:** ACCEPT - Current implementation is acceptable. Temp file leaks unlikely but consider try-finally in future refactoring.
