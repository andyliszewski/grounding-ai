## Identified Risks

### 1. PERF-001: Phrase Detection Performance on Large Scores

**Score: 4 (Medium)**
**Probability**: Medium (2) - Music scores can be very large (hundreds of pages)
**Impact**: Medium (2) - Could slow down processing but not break functionality

**Description**: Musical phrase detection iterates through all elements in a music21 stream. Very large orchestral scores (100+ pages) could result in slow phrase detection.

**Affected Components**:
- `pdf2llm/hybrid_processor.py::detect_phrases()`
- `pdf2llm/hybrid_processor.py::chunk_music()`

**Mitigation Strategy**:
- **Type**: Preventive
- **Actions**:
  - Implement pagination or streaming for very large scores
  - Add timeout mechanism for phrase detection
  - Cache detection results when possible
  - Consider sampling strategy for extremely large works

**Testing Requirements**:
- Performance test with large score (50+ pages)
- Measure phrase detection time
- Verify timeout behavior if implemented

**Residual Risk**: Low - Fallback to measure groups ensures functionality
**Owner**: Dev
**Timeline**: Monitor in production; optimize if needed

### 2. DATA-001: Music Content Detection Accuracy

**Score**: 4 (Medium)
**Probability**: Medium (2) - Staff line detection is heuristic-based
**Impact**: Medium (2) - Misclassification leads to wrong parser selection

**Description**: The `detect_music_content()` function uses visual heuristics to classify pages as text vs music. Complex layouts (text with small musical examples) might be misclassified.

**Affected Components**:
- `pdf2llm/hybrid_processor.py::detect_regions()`
- `pdf2llm/omr_parser.py::detect_music_content()`

**Mitigation Strategy**:
- **Type**: Detective + Corrective
- **Actions**:
  - Log classification confidence scores
  - Implement manual override capability in future
  - Track misclassification rate in telemetry
  - Document known edge cases

**Testing Requirements**:
- Test with hybrid pages (text + small music snippets)
- Verify >80% accuracy target from Epic 7
- Test edge cases: rotated pages, low-res scans

**Residual Risk**: Low - 80% accuracy target from Epic 7, acceptable for MVP
**Owner**: Dev/Product
**Timeline**: Monitor with real-world PDFs
