## Story Description

As a **developer**, I want to **split formatted Markdown into LLM-friendly chunks using LangChain** so that downstream metadata and manifest stages operate on deterministic, size-constrained segments.

This story introduces the chunking layer described in Epic 3. It selects and configures LangChain's `RecursiveCharacterTextSplitter`, exposes tuning knobs (chunk size, overlap), and ensures Markdown structure is respected to avoid cutting tables or headings mid-block.

---

## Context

### From PRD v0.2  
[Source: docs/prd.md — Sections "Chunking Requirements" & "Output Guarantees"]

- Chunks must average ~1,200 characters and honor a 150-character overlap unless overridden.  
- Markdown structure (headings, tables, code blocks) should remain intact to preserve semantic cues.  
- Chunking must be deterministic so QA can diff repeated runs.

### From Architecture v0.2  
[Source: docs/architecture.md — Sections 4 Pipeline Overview & 7 Chunking]

- Chunking runs after formatting and before metadata decoration.  
- The splitter should be configurable through a small `ChunkConfig` object to support CLI overrides introduced later.  
- Chunk order and boundaries must remain stable across runs given identical inputs.

### From Epic 3  
[Source: docs/epics/epic-3-chunking-metadata-v02.md — Story 3.1]

- Adopt LangChain `RecursiveCharacterTextSplitter`.  
- Wire chunk size/overlap configuration.  
- Return a list of chunk strings for metadata processing in Story 3.3.

---

## Acceptance Criteria

### AC1: Splitter Configuration
- ✅ Implement `ChunkConfig` (or similar) with `chunk_size` default 1_200 and `chunk_overlap` default 150.  
- ✅ `chunker.py` exposes `split_markdown(text: str, config: ChunkConfig) -> list[str]`.  
- ✅ `split_markdown` uses LangChain `RecursiveCharacterTextSplitter` under the hood.

### AC2: Structure Preservation
- ✅ Headings, tables, and code blocks are not split mid-block when possible (verified with fixture Markdown).  
- ✅ Overlap material appears at the end of chunk N and the beginning of chunk N+1 without duplication artifacts.  
- ✅ Chunk boundaries are deterministic given the same inputs/config.

### AC3: Configurability & Validation
- ✅ Invalid configs (negative sizes, chunk_size <= chunk_overlap, etc.) raise `ValueError`.  
- ✅ Configs can be supplied via simple dataclass or dict; defaults apply when omitted.  
- ✅ Function logs splitter parameters and resulting chunk counts for observability.

---

## Implementation Guidance

1. **Dependencies:** Add `langchain-text-splitters>=0.0.1` to project dependencies (pyproject/requirements).  
2. **Module:** Create `pdf2llm/chunker.py` with `ChunkConfig` dataclass and `split_markdown` function.  
3. **Splitter Setup:** Instantiate `RecursiveCharacterTextSplitter` with `separators=["\n\n", "\n", " "]` to favor Markdown aware splits and pass config size/overlap.  
4. **Validation:** Guard against invalid configs before instantiating the splitter; include helpful error messages.  
5. **Logging:** Use module logger (`pdf2llm.chunker`) to record chunk count, size, overlap, and sample headings for debugging.  
6. **Future Hooks:** Ensure return type (list of strings) and function signature align with upcoming metadata stories (3.3/3.4).
