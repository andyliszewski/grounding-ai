## Dev Notes

### Module Purpose

This module transforms List[FormulaElement] (from formula_extractor.py) into consumable output formats. Bridges gap between raw extraction and final output files.

**Pipeline Position**:
Formula Extractor → **Formula Formatter** → Writer (file output)

[Source: Epic 8 Story 8.3 requirements]

### LaTeX Format (PRIMARY OUTPUT)

LaTeX is the default format (per Epic 8 decision) because it's the standard for mathematical notation and LLM-compatible.

**Why LaTeX**:
- De facto standard for mathematical typesetting
- LLMs trained on LaTeX notation
- Computational tools (sympy, sage) parse LaTeX
- Human-readable and editable

**Example**:
```latex
$E=mc^2$  % inline equation
$$\int_0^\infty e^{-x^2}dx = \frac{\sqrt{\pi}}{2}$$  % display equation
```

[Source: Epic 8 Technical Architecture]

### MathML Format (SEMANTIC REPRESENTATION)

MathML provides semantic structure for accessibility and computational processing:

**Example**:
```xml
<math xmlns="http://www.w3.org/1998/Math/MathML">
  <mrow>
    <mi>E</mi>
    <mo>=</mo>
    <mi>m</mi>
    <msup><mi>c</mi><mn>2</mn></msup>
  </mrow>
</math>
```

**Use Cases**:
- Accessibility (screen readers)
- Semantic search
- Computational validation

**Implementation**:
Use latex2mathml library:
```python
from latex2mathml.converter import convert
mathml = convert(latex_str)
```

[Source: Epic 8 Story 8.3 requirements]

### Plain Text Fallback (ACCESSIBILITY)

For environments without LaTeX rendering:

**Examples**:
- `E=mc^2` → "E equals m times c squared"
- `\sum_{n=1}^\infty \frac{1}{n^2}` → "sum from n=1 to infinity of 1/n^2"
- `\int_a^b f(x)dx` → "integral from a to b of f(x) dx"

**Limitations**: Complex notation loses structure, best-effort only.

### Markdown Integration

**Inline formulas** (within paragraphs):
```markdown
Einstein showed that $E=mc^2$ relates energy and mass.
```

**Display formulas** (centered, own line):
```markdown
The Gaussian integral:
$$\int_{-\infty}^\infty e^{-x^2}dx = \sqrt{\pi}$$
```

[Source: Epic 8 Output Structure]

### Error Handling

**FormattingError** exception:
```python
class FormattingError(Exception):
    """Raised when formula formatting fails."""
    def __init__(self, format_type: str, formula_id: str, message: str):
        self.format_type = format_type
        self.formula_id = formula_id
        super().__init__(f"{format_type} formatting error for {formula_id}: {message}")
```

**Error Scenarios**:
1. Invalid LaTeX → Log error, skip formula
2. MathML conversion failure → Fallback to LaTeX only
3. Plain text conversion → Best-effort, never fail

[Source: pdf2llm error handling patterns]

### Output File Structure

```
out/<slug>/
  doc.md              # Text with embedded LaTeX
  formulas/
    formula_0001.tex  # Individual formula LaTeX
    formula_0001.mathml
    formula_0001.txt  # Plain text
```

Writer module (Story 8.5) handles file creation.
