**I want** the staging watcher to automatically update agent embeddings after ingesting documents,
**so that** semantic search indexes stay current without manual intervention.

## Acceptance Criteria

1. Watcher triggers incremental embedding update after successful document ingestion
2. Determines affected agents based on document collections
3. Lock file prevents concurrent embedding updates
4. Configurable via environment variable to enable/disable auto-embedding
5. Logs embedding update status and timing
6. Handles embedding failures gracefully (doesn't block document ingestion)
7. Systemd service updated with embedding configuration

### Agent Detection Logic

```bash
# Pseudocode for finding affected agents
find_affected_agents() {
    local collection="$1"
    local affected=()

    for agent_file in "$AGENTS_DIR"/*.yaml; do
        # Parse YAML for corpus_filter.collections
        if grep -q "collections:" "$agent_file"; then
            if grep -A10 "collections:" "$agent_file" | grep -q "$collection"; then
                affected+=("$(basename "$agent_file" .yaml)")
            fi
        fi
    done

    echo "${affected[@]}"
}
```

### Lock File Pattern

```bash
LOCK_FILE="$EMBEDDINGS_DIR/_embeddings.lock"
LOCK_TIMEOUT=3600  # 1 hour

acquire_lock() {
    if [[ -f "$LOCK_FILE" ]]; then
        local lock_age=$(($(date +%s) - $(stat -c %Y "$LOCK_FILE")))
        if [[ $lock_age -gt $LOCK_TIMEOUT ]]; then
            log "Stale lock detected (${lock_age}s old), removing"
            rm -f "$LOCK_FILE"
        else
            log "Embedding update skipped: lock held"
            return 1
        fi
    fi
    echo "$$" > "$LOCK_FILE"
    return 0
}

release_lock() {
    rm -f "$LOCK_FILE"
}
```

### Integration Point

In `process_collection()` after successful processing:

```bash
# After pdf2llm completes successfully
if [[ "${AUTO_EMBEDDINGS:-false}" == "true" ]]; then
    trigger_embedding_update "$collection"
fi
```
