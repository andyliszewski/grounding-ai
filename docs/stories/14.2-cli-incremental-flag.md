## Story

**As a** corpus administrator,
**I want** an `--incremental` flag on the `pdf2llm embeddings` command,
**so that** I can update agent embeddings quickly after adding new documents.

---

## Acceptance Criteria

1. `pdf2llm embeddings --incremental` flag added to CLI
2. Incremental mode skips documents already in chunk map
3. Only processes new/updated documents detected by staleness check
4. Tombstones deleted documents automatically
5. Falls back to full rebuild if index doesn't exist
6. `--check` flag enhanced to show incremental update recommendation
7. Progress output shows "X new, Y deleted, Z updated" statistics

## Dev Notes

### CLI Usage Examples

```bash
# Full embedding generation (existing behavior)
pdf2llm embeddings --agent scientist --corpus ./corpus

# Incremental update (new)
pdf2llm embeddings --agent scientist --corpus ./corpus --incremental

# Check what would be updated
pdf2llm embeddings --agent scientist --corpus ./corpus --check
```

### Expected --check Output

```
=== Embedding Staleness Check: scientist ===

Index: embeddings/scientist/_embeddings.faiss
  Vectors: 167,581
  Tombstones: 19 (0.01%)
  Created: 2026-01-08T10:00:00Z
  Updated: 2026-01-09T14:30:00Z

Changes detected:
  New documents: 3
  Deleted documents: 0
  Updated documents: 1

Recommendation: Use --incremental (estimated <2 min vs ~3.5h full rebuild)
```

### Dependencies on Story 14.1

- `check_index_staleness()` - Detect what needs updating
- `append_to_vector_index()` - Add new embeddings
- `tombstone_documents()` - Mark deletions
- `StalenessReport` dataclass - Structure for change detection
