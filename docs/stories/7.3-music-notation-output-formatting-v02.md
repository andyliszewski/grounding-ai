## Dev Notes

### Module Purpose

This module transforms List[MusicElement] (from omr_parser.py) into consumable output formats. It bridges the gap between raw OMR extraction and final output files.

**Pipeline Position**:
OMR Parser → **Music Formatter** → Writer (file output)

[Source: Epic 7 Story 7.3 requirements]

### music21 Library Usage

music21 is the core library for music notation manipulation:

```python
from music21 import stream, note, clef, key, meter

# Create a stream (music21's container for musical elements)
s = stream.Stream()
s.append(clef.TrebleClef())
s.append(key.KeySignature(0))  # C major (0 sharps/flats)
s.append(meter.TimeSignature('4/4'))
s.append(note.Note('C4', quarterLength=1.0))

# Export to various formats
musicxml_str = s.write('musicxml')
abc_str = s.write('abc')
s.write('midi', fp='output.mid')
```

[Source: music21 documentation, Epic 7 dependencies]

### MusicXML Format (PRIMARY OUTPUT)

MusicXML is the default format (per Epic 7 decision) because it's the most semantically rich for agent analysis.

**Why MusicXML**:
- Contains explicit pitch, duration, key, time signature metadata
- Preserves harmonic structure, voice leading, articulations
- Industry standard - works with music notation software (Finale, Sibelius, MuseScore)
- music21 provides robust MusicXML export

**Validation**:
Use music21's built-in validation or lxml schema validation:
```python
from music21.musicxml import m21ToXml
exporter = m21ToXml.GeneralObjectExporter(s)
musicxml_bytes = exporter.parse()
# Optionally validate against MusicXML 3.1 XSD schema
```

[Source: Epic 7 Decisions Made - Default Music Format]

### ABC Notation Format (LIGHTWEIGHT ALTERNATIVE)

ABC is a text-based format, human-readable:

**Example ABC**:
```
X:1
T:Simple Melody
M:4/4
L:1/4
K:C
CDEF|GABc|
```

**When to use**:
- Web display (ABC.js can render in browser)
- Lightweight transmission
- Human-readable notation

**Validation**:
Call abc2midi to verify ABC syntax:
```python
result = subprocess.run(['abc2midi', 'test.abc', '-o', 'test.mid'], capture_output=True)
if result.returncode != 0:
    raise ABCFormatError("Invalid ABC notation")
```

[Source: Epic 7 Use Cases - Music Education Materials]

### MIDI Format (PLAYBACK)

MIDI is for audio playback, not semantic analysis:

**Use Cases**:
- Verify notation extraction by listening
- Playback in DAWs or MIDI players
- Convert to audio via synthesizers

**Limitations**:
- Loses notation semantics (no key signatures, enharmonic spellings ambiguous)
- Not suitable for music theory analysis

**Generation**:
music21 handles MIDI export automatically:
```python
s.write('midi', fp='output.mid')
```

[Source: Epic 7 Goals - Multiple output formats]

### Error Handling

**FormattingError** exception for all formatting failures:

```python
class FormattingError(Exception):
    """Raised when music notation formatting fails."""
    def __init__(self, format_type: str, message: str):
        self.format_type = format_type
        super().__init__(f"{format_type} formatting error: {message}")
```

**Error Scenarios**:
1. Empty music_elements → Return minimal valid output or raise error
2. Invalid MusicElement data → Log which element failed, skip it
3. music21 export failure → Raise FormattingError with music21 error details
4. ABC/MIDI validation failure → Include validation output in error message

[Source: pdf2llm error handling patterns]

### Output File Naming

Following pdf2llm output structure:

```
out/<slug>/
  music.musicxml  (default, always generated)
  music.abc       (if --music-format abc or all)
  music.mid       (if --music-format midi or all)
  music_metadata.md  (markdown summary)
```

Writer module (Story 7.5) will handle file creation. This module only generates content strings/bytes.

[Source: docs/architecture.md - Output Structure, Epic 7 Story 7.5]
