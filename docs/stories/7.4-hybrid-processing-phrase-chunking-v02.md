## Story

**As a** pdf2llm developer,
**I want** to process documents containing both text and music notation while chunking music by semantic phrases,
**so that** music theory textbooks and educational materials are correctly parsed with musically meaningful chunk boundaries.

5. **Phrase-based chunking** creates semantic musical units

7. Combined output preserves original document sequence
8. Metadata clearly labels each region type (text/music) and phrase boundaries
9. Hybrid documents generate both markdown text and music notation files
10. Integration tests validate hybrid processing and phrase chunking with sample PDFs

## Dev Notes

### Hybrid Document Use Case

**Primary Use Case**: Music theory textbooks with alternating text explanations and music notation examples.

**Example Document Structure**:
- Page 1: Text explaining harmonic progression
- Page 2: Music notation examples of I-IV-V progression
- Page 3: Text explaining voice leading rules
- Page 4: Music examples of voice leading

**Processing Goal**:
- Extract text as markdown chunks
- Extract music as MusicXML chunks
- Preserve sequence for LLM context (text explanation → music example → next text)

[Source: Epic 7 Overview, Use Cases]

### Musical Phrase Concept

**What is a Musical Phrase?**
A musical phrase is a semantic unit of music, analogous to a sentence in language. Typical characteristics:
- 4-8 measures long (in common practice music)
- Ends with a cadence (harmonic resolution)
- Often marked by rests, breath marks, or slurs
- Represents a complete musical thought

**Why Phrase-Based Chunking?**
Per Epic 7 user decision: Phrases are semantic musical units vs mechanical measures. This makes chunks musically meaningful for agent analysis.

**Examples**:
- "Mary Had a Little Lamb" = 1 phrase (8 measures)
- Beethoven's 5th Symphony opening = 2 phrases (4 + 4 measures)

[Source: Epic 7 Decisions Made - Chunking Music Notation, music theory principles]

### music21 Phrase Detection

**Primary Method**: music21's built-in phrase analysis:

```python
from music21 import converter, analysis

stream = converter.parse('music.musicxml')

# Method 1: Phrase detection with rests
phrases = stream.analyze('phraseWithRests')
# Returns: List of phrases based on rest positions

# Method 2: Cadence detection
for measure in stream.flat.measures:
    analysis_result = measure.analyze('key')
    if hasattr(analysis_result, 'cadence'):
        # Cadence detected → phrase boundary
```

**Detection Signals** (in priority order):
1. Explicit phrase markings (if present in score)
2. Rests >= quarter note duration
3. Harmonic cadences (V→I, IV→I)
4. Slur endings
5. Dynamic/tempo changes

[Source: music21 documentation, Epic 7 Story 7.4 requirements]

### Fallback Chunking Strategies

**Hierarchy** (when phrase detection fails):

1. **System-based chunking**:
   - System = one line of music on the printed page
   - Typically 1-4 measures per system
   - Detected from visual layout in PDF
   - Preserves visual grouping

2. **Measure-group chunking**:
   - Chunk every 4 or 8 measures (common phrase lengths)
   - Simple, deterministic fallback
   - May split actual phrases but better than no chunking

**Implementation**:
```python
def chunk_music(stream):
    # Try phrase detection first
    phrases = detect_phrases(stream)
    if phrases and len(phrases) > 0:
        return chunk_by_phrases(phrases)

    # Fallback to systems
    systems = detect_systems(stream)
    if systems:
        return chunk_by_systems(systems)

    # Final fallback to measure groups
    return chunk_by_measure_groups(stream, group_size=4)
```

[Source: Epic 7 Story 7.4 requirements - Fallback chunking]

### Region Detection Approach

**Whole-Page Strategy** (simpler, for MVP):
- Classify each page as "text" or "music" using detect_music_content()
- No sub-page bounding boxes
- Simpler implementation, faster processing

**Mixed-Page Strategy** (future enhancement):
- Detect music regions within pages (bounding boxes)
- Extract regions separately
- More complex but handles inline music examples

**MVP Recommendation**: Start with whole-page strategy, add mixed-page in future epic.

[Source: Story 7.2 detect_music_content() function, Epic 7 known limitations]
