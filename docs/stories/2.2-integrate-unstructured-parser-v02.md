## Story Description

As a **developer**, I want to **wrap Unstructured's PDF partitioner behind a thin adapter** so that the pipeline can extract structured elements (headings, tables, paragraphs) while encapsulating OCR and error handling policies defined in Epic 2.

This story implements the parsing module that converts file paths from the scanner into Unstructured element lists. It must expose a clear API, respect configurable OCR modes, and prevent single-file failures from aborting the batch.

---

## Context

### From PRD v0.2  
[Source: docs/prd.md — Sections "Parsing Requirements" & "Reliability"]

- Parsing must preserve document structure (headings, tables, narratives).  
- OCR should be configurable (auto/on/off) and default to `auto`.  
- Parsing failures should be logged and skipped without halting the entire run.

### From Architecture v0.2  
[Source: docs/architecture.md — Sections 3 Components & 8 Error Handling]

- `parser.py` encapsulates Unstructured integration and exposes `parse_pdf`.  
- Error handling strategy: log and continue on failure, record metadata for skipped files.  
- OCR decisions influence downstream formatting and must be observable via logging.

### From Epic 2  
[Source: docs/epics/epic-2-parser-integration-v02.md — Story 2.2]

- Wrap `unstructured.partition_pdf()` with configuration for OCR and table extraction.  
- Return structured element lists for downstream formatting.  
- Catch exceptions, log details, and propagate a recoverable signal.

## Acceptance Criteria

### AC1: Parser API
- ✅ `parse_pdf(file_path: Path, ocr_mode: str = "auto") -> list[Any]` defined in `pdf2llm/parser.py`.  
- ✅ Accepts `Path` objects and performs type validation.  
- ✅ Returns `list[Element]` (as produced by Unstructured) or raises controlled exceptions.

### AC2: OCR Strategy Handling
- ✅ Supports modes `auto`, `on`, `off`.  
- ✅ `auto` triggers OCR only when Unstructured reports low text density.  
- ✅ Invalid OCR mode raises `ValueError` with valid options listed.  
- ✅ OCR decisions logged with file name and mode used.

### AC3: Unstructured Integration
- ✅ Calls `partition_pdf` with parameters: `infer_table_structure=True`, `extract_images_in_pdf=False`, `ocr_languages="eng"`.  
- ✅ Passes file path via `filename=str(file_path)` to keep lazy evaluation.  
- ✅ Provides hook for future configuration (e.g., metadata).

### AC4: Error Handling
- ✅ Catches `Exception` from Unstructured, logs stack trace at debug level, and returns `None` or raises a custom `ParseError` that the pipeline can catch.  
- ✅ Ensures known errors (e.g., password-protected PDF) surface clear messages.  
- ✅ Does not leave partial state (no temporary files left behind).

### AC5: Telemetry & Metrics
- ✅ Emits structured log message (JSON-friendly dict) summarizing pages parsed, OCR usage, and elapsed time.  
- ✅ Integrates with existing logging configuration (`logging.getLogger("pdf2llm.parser")`).

## Implementation Guidance

1. **Create Module:** Add `pdf2llm/parser.py` with public function `parse_pdf`.  
2. **Validate Inputs:** Ensure `file_path` exists and is a file before invoking Unstructured.  
3. **Map OCR Modes:**  
   - `off` → `ocr_strategy="never"`  
   - `on` → `ocr_strategy="always"`  
   - `auto` → start with `ocr_strategy="auto"` and rely on Unstructured heuristics.  
4. **Invoke Unstructured:**  
   ```python
   elements = partition_pdf(
       filename=str(file_path),
       strategy="auto",
       infer_table_structure=True,
       extract_images_in_pdf=False,
       ocr_languages="eng",
       ocr_strategy=resolved_strategy,
   )
   ```  
5. **Telemetry:** Measure runtime using `time.perf_counter()` and log summary data.  
6. **Error Wrapper:** Define `class ParseError(Exception)` to carry filename and root cause; raise when recoverable failure occurs.  
7. **Documentation:** Include module-level docstring referencing Epic 2 and OCR policy.
