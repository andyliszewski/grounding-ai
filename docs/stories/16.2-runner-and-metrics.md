## Story

**As a** grounding-ai contributor,
**I want** to execute fixture queries against an agent's FAISS index and compute standard retrieval metrics,
**so that** retrieval changes (chunking, embeddings, reranking, hybrid) produce comparable, defensible numbers.

---

## Acceptance Criteria

1. `grounding/eval/runner.py` exposes `run_eval(fixture_set, agent_name, top_k=10) -> EvalRun` that executes each fixture query against the agent's FAISS index and returns per-item and aggregate results.
2. `grounding/eval/metrics.py` provides pure functions: `recall_at_k(retrieved, expected, k)`, `mrr(retrieved, expected)`, `ndcg_at_k(retrieved, expected, k)`. No I/O; deterministic.
3. The runner reuses the existing search path (`grounding/vector_store.py` + `grounding/embedder.py`); no duplicated retrieval logic.
4. Per-item output records: `item_id`, `query`, `expected_doc_ids`, `retrieved_top_k` (list of `{doc_id, chunk_id, score, rank}`), `first_hit_rank` (or `None`), `tags`.
5. Aggregate output records: `recall@1`, `recall@3`, `recall@5`, `recall@10`, `MRR`, `nDCG@10`, plus per-tag breakdown of the same metrics.
6. Hit semantics: a retrieved chunk counts as a hit if its `doc_id` is in `expected.doc_ids` (any-of). When the fixture also lists `expected.chunk_ids`, the runner additionally records `strict_first_hit_rank` for chunk-level matches; aggregate metrics use the doc-level definition by default.
7. Runner warns (logs WARNING, continues) when an `expected.doc_ids` entry is no longer present in the corpus manifest. Aggregate output records the count of skipped items.
8. Unit tests cover metrics with synthetic inputs (no FAISS) and runner with a mocked search function.
9. One integration test runs the runner against a tiny in-repo test corpus committed under `tests/eval_fixtures/` and asserts non-trivial scores.
10. Total test runtime for this story's tests stays under 10s.

## Dev Notes

### Reuse, Don't Reimplement

The runner must call into existing code paths:

| Need | Existing module | Verified signature |
|------|-----------------|--------------------|
| Resolve agent corpus filter | `grounding/agent_filter.py` | `load_agent_config(agent_name, agents_dir) -> AgentConfig` |
| Load FAISS index + chunk map | `grounding/vector_store.py` | `load_vector_index(corpus_dir: Path) -> (faiss.Index, dict)` — pass the per-agent embeddings dir (e.g. `embeddings/<agent>/`) |
| Embed query | `grounding/embedder.py` | use existing query-embedding path (single sentence-transformer call) |
| Similarity search | `grounding/vector_store.py` | `search_similar_chunks(index, chunk_map, query_embedding, top_k) -> List[Tuple[str, float]]` — returns `(chunk_id, distance)` only |
| Read corpus manifest | `grounding/manifest.py` | `ManifestManager.load(corpus_dir)` |

**Doc-id derivation (decided):** `search_similar_chunks` returns `(chunk_id, distance)` and does **not** surface `doc_id`. Do not modify `vector_store.py`. The `chunk_map["chunks"]` list (v1.1+) carries `doc_id` per entry, keyed by `embedding_index`. Build a `chunk_id → doc_id` lookup from the loaded chunk_map at runner init, and resolve doc_id locally for each search result. This keeps the change scoped to `grounding/eval/`.

### Hit Semantics (canonical)

```python
def first_hit_rank(retrieved: list[RetrievedChunk], expected_doc_ids: set[str]) -> int | None:
    for r in retrieved:
        if r.doc_id in expected_doc_ids:
            return r.rank   # 1-indexed
    return None
```

```python
def strict_first_hit_rank(retrieved, expected_chunk_ids: set[str]) -> int | None:
    if not expected_chunk_ids:
        return None
    for r in retrieved:
        if f"{r.doc_id}/{r.chunk_id}" in expected_chunk_ids:
            return r.rank
    return None
```

### Metrics

```python
def recall_at_k(first_hit_ranks: list[int | None], k: int) -> float:
    if not first_hit_ranks:
        return 0.0
    return sum(1 for r in first_hit_ranks if r is not None and r <= k) / len(first_hit_ranks)

def mrr(first_hit_ranks: list[int | None]) -> float:
    if not first_hit_ranks:
        return 0.0
    return sum(1.0 / r for r in first_hit_ranks if r is not None) / len(first_hit_ranks)
```

`nDCG@k` uses binary relevance (1 if `doc_id ∈ expected.doc_ids`, else 0) with standard log2 discount. Document the formula in the module docstring.

### Per-Tag Breakdown

An item with `tags: [methodology, transformers]` contributes to both groups. Output shape:

```python
per_tag = {
    "methodology": {"recall@5": 0.80, "mrr": 0.62, "n_items": 5, "low_sample": False},
    "transformers": {"recall@5": 0.50, "mrr": 0.33, "n_items": 1, "low_sample": True},
}
```

### Skipped Items

If `expected.doc_ids = ["7a9b2c1f"]` but `7a9b2c1f` is not in the corpus manifest, the runner:
1. Logs `WARNING: fixture item ds-003 references unknown doc_id 7a9b2c1f; skipping`.
2. Adds `ds-003` to `EvalRun.skipped`.
3. Excludes the item from aggregate metrics (does not count as a miss).

This prevents corpus churn from silently degrading scores.
