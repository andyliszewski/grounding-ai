## Story Description

As a **developer**, I want a **deterministic scanner that enumerates PDFs in the input directory** so that the parsing pipeline can operate on a well-defined batch of documents and preserve reproducible ordering across runs.

This story creates the lightweight discovery layer that feeds the parser and formatter stages described in Epic 2. The scanner must be resilient to empty folders and unexpected file types while producing stable, case-insensitive results.

---

## Context

### From PRD v0.2  
[Source: docs/prd.md — Processing Requirements]

- Batch processing expects the CLI to accept an input directory and enumerate artifacts automatically.  
- Deterministic outputs are critical for predictable chunking and manifests.  
- The system must gracefully handle missing or malformed inputs without aborting the batch.

### From Architecture v0.2  
[Source: docs/architecture.md — Sections 4 Pipeline Overview & 8 Error Handling]

- `pipeline.scan_inputs()` is the first stage and supplies `Path` objects downstream.  
- Failures in scanner logic must not crash the pipeline; errors should surface as structured logs.  
- Consistent ordering simplifies manifest generation and regression testing.

### From Epic 2  
[Source: docs/epics/epic-2-parser-integration-v02.md — Story 2.1]

- Discover `.pdf` files (case-insensitive).  
- Return a sorted list of `Path` objects.  
- Handle empty directories gracefully.  
- Raise descriptive errors for invalid directories.

## Acceptance Criteria

### AC1: Directory Validation
- ✅ Non-existent input directories raise a `FileNotFoundError` with the path in the message.  
- ✅ Non-directory inputs (files) raise a `NotADirectoryError`.  
- ✅ Valid directories proceed without side effects (no writes).

### AC2: PDF Discovery & Filtering
- ✅ All files ending with `.pdf` or `.PDF` (case-insensitive) are included.  
- ✅ Non-PDF files are ignored without raising.

### AC3: Deterministic Ordering
- ✅ Returned paths are sorted alphabetically using `Path.name.lower()` to ensure case-stable ordering.  
- ✅ Sorting is stable across OS filesystems.

### AC4: Graceful Empty Handling
- ✅ Empty directories return an empty list.  
- ✅ Function logs an informational message when zero PDFs are found (hook via logger stub for now).

### AC5: API Contract
- ✅ Scanner returns `List[Path]`.  
- ✅ Function signature aligns with `pipeline.py` expectations (e.g., `def scan_pdfs(input_dir: Path) -> list[Path]:`).  
- ✅ Module placed in `pdf2llm/scanner.py` with clear docstring referencing Epic 2.

---

## Implementation Guidance

1. **Create Module:** Add `pdf2llm/scanner.py` with a pure function `scan_pdfs(input_dir: Path) -> list[Path]`.  
2. **Validate Input:** Use `pathlib.Path` methods (`exists()`, `is_dir()`) for guard clauses.  
3. **Enumerate PDFs:** Iterate with `iterdir()` or `glob("*.pdf")` while applying `casefold()` for extension checks.  
4. **Sort Paths:** Sort using `sorted(paths, key=lambda p: p.name.casefold())` to normalize case sensitivity.  
5. **Logging Hook:** Use the existing logging facility (or placeholder `logger = logging.getLogger(__name__)`) to emit discovery metrics.  
6. **Docstring:** Reference Epic 2 and describe deterministic ordering requirement for future maintainers.
