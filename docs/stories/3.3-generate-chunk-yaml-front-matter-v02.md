## Story Description

As a **developer**, I want to **decorate each Markdown chunk with consistent YAML front matter** so that downstream consumers (vector stores, manifests) receive rich metadata describing document lineage, positions, and hashes.

This story builds on chunk outputs (Story 3.1) and doc IDs (Story 3.2) to attach structured metadata to every chunk. It defines the YAML schema, generates chunk identifiers, and ensures timestamps and hashes are recorded in a machine-readable way.

---

## Context

### From PRD v0.2  
[Source: docs/prd.md — Sections "Metadata Requirements" & "Chunk Contracts"]

- Each chunk must include doc ID, chunk index, page range, and content hash.  
- Metadata should be parseable via YAML and consistent across runs.  
- Created timestamps use ISO8601 UTC format for auditability.

### From Architecture v0.2  
[Source: docs/architecture.md — Sections 6 Output Writer & 7 Chunking]

- Chunk metadata is assembled prior to writing outputs; writer simply persists front matter + body.  
- `ChunkMetadata` structure should capture doc/context info and be serializable.  
- The pipeline tracks page numbers / section context from parsing stage when available.

### From Epic 3  
[Source: docs/epics/epic-3-chunking-metadata-v02.md — Story 3.3]

- Include fields: `doc_id`, `source`, `chunk_id`, `page_start`, `page_end`, `hash`, `created_utc`.  
- YAML must be valid and parseable by downstream pipelines.  
- Combine front matter and chunk content into final string.

---

## Acceptance Criteria

### AC1: Metadata Schema
- ✅ Define `ChunkMetadata` dataclass (or similar) with required fields plus optional `section_heading`.  
- ✅ Provide helper to map pipeline `FileContext` + chunk index to metadata instance.  
- ✅ `chunk_id` composed of `f"{doc_id}-{index:04d}"`.

### AC2: YAML Serialization
- ✅ Implement `render_chunk(metadata: ChunkMetadata, content: str) -> str` returning front matter + content separated by blank line.  
- ✅ Use lowercase boolean / ISO8601 UTC timestamps (`datetime.utcnow().replace(tzinfo=timezone.utc).isoformat()`).  
- ✅ Ensure YAML keys follow snake_case and consistent ordering for determinism.

### AC3: Hash & Page Data Integration
- ✅ `metadata.hash` generated via Story 3.4 helper (content hash without YAML).  
- ✅ `page_start`/`page_end` derived from parser metadata when available; fallback to `None` or chunk index if absent.  
- ✅ Module logs when page info is missing (debug-level) for later visibility.

---

## Implementation Guidance

1. **Module:** Create `pdf2llm/chunk_metadata.py` exposing `ChunkMetadata` and `render_chunk`.  
2. **Dependencies:** Reuse hashing helper from Story 3.4 (or temporary stub if sequenced earlier).  
3. **Timestamp:** Generate once per chunk render to avoid inconsistent times within a batch; consider allowing injection (useful for tests).  
4. **Determinism:** Sort YAML keys manually or via ordered dataclass fields; avoid relying on `yaml.dump` to minimize dependencies.  
5. **Integration:** Hook into pipeline after chunking so metadata outputs feed writer in future epics.
