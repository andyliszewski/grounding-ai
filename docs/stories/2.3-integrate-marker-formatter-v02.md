## Story Description

As a **developer**, I want to **convert parsed Unstructured elements into clean Markdown using Marker** so that pdf2llm emits structured, LLM-ready documents with preserved headings, tables, and lists.

This story builds the formatter layer that accepts either raw PDF paths (for direct Marker use) or the element list returned from Story 2.2. It must normalize output, capture conversion metadata, and shield the pipeline from conversion failures.

## Acceptance Criteria

### AC1: Formatter API
- ✅ `format_markdown(elements: Sequence[Any], *, metadata: dict | None = None) -> str` defined in `pdf2llm/formatter.py`.  
- ✅ Accepts Unstructured elements or raw PDFs (via overload helper) and returns UTF-8 Markdown string.  
- ✅ Raises `FormatError` (custom) on unrecoverable failures.

### AC2: Marker Integration
- ✅ Invokes `marker.convert.convert_elements` (or equivalent API) using parameters aligned with Epic 2 assumptions.  
- ✅ Ensures tables remain in Markdown table syntax and headings map to `#`, `##`, etc.  
- ✅ Supports optional `metadata` dict to inject front matter hints for writer stage.

### AC3: Deterministic Output
- ✅ Applies post-processing to normalize line endings (`\n`) and trim trailing whitespace.  
- ✅ Ensures repeated invocations with identical inputs yield identical strings (verified in tests).

### AC4: Error Handling & Logging
- ✅ Catches Marker-specific exceptions, logs context (file slug, element count), and raises `FormatError`.  
- ✅ Emits warning-level logs when falling back to passthrough (plain text) if Marker fails and fallback enabled.  
- ✅ Records conversion duration for observability via log field `formatter_duration_ms` on the formatter logger.

### AC5: Fallback Strategy (MVP)
- ✅ Provide optional flag `allow_plaintext_fallback: bool = False`.  
- ✅ When enabled and Marker fails, emit Markdown with YAML front matter `fallback: true` and a body built from element text gathered via `getattr(element, "text", "")`, separated by blank lines.

---

## Implementation Guidance

1. **Create Module:** Add `pdf2llm/formatter.py` with `format_markdown` and optional helper `format_pdf(file_path: Path, **kwargs)`.  
2. **Normalize Inputs:** If given file path, call Marker’s PDF helper; otherwise, convert element list using supported API.  
3. **Post-processing:**  
   - Use `textwrap.dedent` or manual normalization to remove redundant indentation.  
   - Replace Windows line endings with `\n`.  
4. **Fallback Handling:**  
   - Detect failure via `except Exception as exc`.  
   - If fallback enabled, build fallback Markdown that adds front matter `fallback: true`, joins element text via `getattr(element, "text", "")` (skipping empty strings), and logs a warning with `fallback_used=true`.  
5. **Telemetry:** Use module-level logger to capture `formatter_duration_ms`, `element_count`, `fallback_used`.  
6. **Docstrings:** Reference Epic 2, list dependencies (`marker`) for maintainers.
