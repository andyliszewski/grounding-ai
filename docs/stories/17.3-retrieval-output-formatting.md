## Story

**As a** user querying a grounded agent,
**I want** retrieved chunks shown to the LLM to carry a compact citation prefix (slug, page, section),
**so that** the model can produce precise citations like "per alpha-paper, p.247, §3.2 Bootstrap Methods" instead of vague document references — and users can verify claims without scrolling.

---

## Acceptance Criteria

1. A new module-level helper `format_citation_prefix(source, page_start, page_end, section_heading) -> str` lives in a shared location usable by both `scripts/search_corpus_tool.py` and `mcp_servers/corpus_search/server.py`. Recommend placing it at `grounding/citations.py` so it ships with the installed package.
2. Format rules:
   - Full: `[<slug>, p.<start>, §<section>]` when all fields present and `start == end`.
   - Range: `[<slug>, p.<start>–<end>, §<section>]` when `start != end`.
   - Page-only: `[<slug>, p.<start>[–<end>]]` when section is null.
   - Section-only: `[<slug>, §<section>]` when pages are null.
   - Slug-only: `[<slug>]` when both pages and section are null.
   - Uses en-dash (`–`, U+2013) between pages, not hyphen.
   - `<slug>` is the document slug (derived from `source` via existing slugify, or read directly when available).
3. `scripts/search_corpus_tool.py`:
   - `_read_chunk` extracts `page_start`, `page_end`, `section_heading` from the YAML front matter.
   - `_format_results` prefixes each result's body with `format_citation_prefix(...)` on its own line, immediately after the existing `--- Result N (source: ...) ---` header.
   - Existing `source`, `score`, `chunk_id`, `doc_id` fields continue to appear in the per-result dict.
4. `mcp_servers/corpus_search/server.py`:
   - `read_chunk` surfaces the three new metadata fields.
   - `format_results_for_context` prefixes each chunk block with the citation prefix.
   - Existing trailing `*doc_id: X, chunk: Y*` annotation is preserved.
5. The citation prefix is emitted on its **own line** (leading), separable by consumers that parse raw chunk text. Body text is not mutated.
6. Chunks missing the new fields (pre-17.2 on-disk content, pdftotext fallback) degrade to whichever prefix form fits — no crash, no warnings at WARNING level, and no naked brackets like `[slug, , ]`.
7. `CLAUDE.md`'s "Metadata Contracts" section gains a "Retrieval Output Format" sub-section documenting the citation prefix format and semantics.
8. Unit tests cover all five prefix variants plus null-field combinations. Integration tests confirm the prefix appears end-to-end for both the local agentic tool and the MCP server paths.
9. Epic 16 mini-corpus eval gate stays green on this PR (no retrieval regression; the prefix is additive, not replacing retrieval).
10. No new runtime dependencies.

### Slug derivation

`source` in chunk front matter is the original filename (e.g., `alpha-paper.pdf`). The prefix should show the *slug* (e.g., `alpha-paper`), matching how the corpus directory is named. Derivation:
1. If `source` looks like a filename (has an extension), strip the extension and slugify the stem.
2. If `source` is already a slug (no extension, matches kebab-case pattern), use as-is.
3. Document this in the `format_citation_prefix` docstring.

If a cleaner `slug` field is available upstream (e.g., from `meta.yaml`), prefer it — but do not add new I/O to the formatter for this. Keep the helper pure.

### Placement decision: `grounding/citations.py`

Placing the helper in the installed package (not under `scripts/`) makes it reachable from:
- Both current retrieval paths (`scripts/search_corpus_tool.py`, `mcp_servers/corpus_search/server.py`)
- Future paths (e.g., a web UI if Tier 3 #12 ships)
- Eval harness (Story 17.4 may use it when computing `citation_accuracy`)

No existing `grounding/citations.py`; this is a greenfield small module.

### Out of scope

- Changing how the *LLM prompt* instructs the model to cite. That's a separate concern (system prompts, agent personas). This story only ensures the *data* carries citations.
- Hyperlinking the citation to a source doc or page (future UI work).
- Figure/table/footnote citations.
