## Semantic Search with Embeddings

### Enable Semantic Search

Generate vector embeddings during ingestion for semantic search capabilities:

\`\`\`bash
pdf2llm --in ./textbooks \\
        --out ./corpus \\
        --parser unstructured \\
        --emit-embeddings
\`\`\`

This creates two additional files in the corpus:
- `_embeddings.faiss`: Vector index for similarity search
- `_chunk_map.json`: Mapping of chunk IDs to embeddings

### Query the Corpus

Search for relevant content using natural language:

\`\`\`bash
python -m pdf2llm.query \\
    --corpus ./corpus \\
    --query "What is the stress-strain relationship?" \\
    --top-k 5
\`\`\`

### Complete Workflow Example

\`\`\`bash
# 1. Ingest PDFs with embeddings
pdf2llm --in ./research-papers \\
        --out ./research-corpus \\
        --parser unstructured \\
        --emit-embeddings \\
        --clean

# 2. Query for relevant chunks
python -m pdf2llm.query \\
    --corpus ./research-corpus \\
    --query "machine learning optimization techniques" \\
    --top-k 10

# 3. Query with JSON output for programmatic use
python -m pdf2llm.query \\
    --corpus ./research-corpus \\
    --query "neural networks" \\
    --top-k 5 \\
    --format json
\`\`\`

### Agent Integration

Use with LLM agents for intelligent question answering:

\`\`\`python
# In your agent instructions:
"""
When answering questions:
1. Query the corpus: python -m pdf2llm.query --corpus ./corpus --query "{question}" --top-k 5
2. Read the returned chunks
3. Synthesize answer from the content
4. Cite sources using chunk IDs
"""
\`\`\`

### Performance

- **Ingestion with embeddings**: ~10% slower than without
- **Embedding generation**: >100 chunks/second on CPU
- **Query latency**: <100ms for top-10 results
- **Memory overhead**: ~1.5KB per chunk

### Dependencies

Requires additional packages:
\`\`\`bash
pip install sentence-transformers>=2.2.0 faiss-cpu>=1.7.0
\`\`\`

Model download (first run only): ~80MB

### Troubleshooting Embeddings

**"Vector index not found"**: Re-run ingestion with `--emit-embeddings`

**Model download fails**: Check internet connection (first run only)

**Slow embedding generation**: Normal on CPU; GPU version available via `faiss-gpu`

**Out of memory**: Reduce batch size or upgrade RAM for very large corpora
\`\`\`

## 13. Vector Embeddings & Semantic Search (v0.2.1)

### Overview

Optional semantic search capability using sentence-transformers embeddings and FAISS vector store.

### Architecture

\`\`\`
Enhanced Pipeline:
Parse → Format → Chunk → Metadata → [Embed] → Write → [Vector Store]
                                      ^^^^^^      ^^^^^^
                                      NEW STEPS

Query Flow:
User Query → Generate Embedding → FAISS Search → Load Chunks → Return Results
\`\`\`

### Components

**embedder.py**: Embedding generation using sentence-transformers
- Model: all-MiniLM-L6-v2 (384 dimensions)
- Performance: ~1000 chunks/second on CPU
- Deterministic outputs

**vector_store.py**: FAISS index management
- Index type: Flat L2 (exact search)
- Persistence: _embeddings.faiss, _chunk_map.json
- Scalability: 100k+ chunks

**query.py**: Semantic search interface
- CLI: python -m pdf2llm.query
- API: query_corpus(corpus, query, top_k)
- Output: ranked chunks with scores

### Dependencies

\`\`\`python
sentence-transformers>=2.2.0  # Embedding model
faiss-cpu>=1.7.0              # Vector similarity search
torch>=2.0.0                  # Required by sentence-transformers
\`\`\`

### Usage

\`\`\`bash
# Ingest with embeddings
pdf2llm --in ./pdfs --out ./corpus --emit-embeddings

# Query corpus
python -m pdf2llm.query --corpus ./corpus --query "..." --top-k 5
\`\`\`

### Performance Characteristics

- Embedding generation: >100 chunks/sec
- Query latency: <100ms for top-10
- Memory overhead: ~1.5KB per chunk
- Disk overhead: ~1.5KB per chunk

### Future Enhancements

- GPU acceleration (faiss-gpu)
- Alternative vector stores (Chroma, Pinecone)
- Approximate nearest neighbor (ANN) indices
- Incremental index updates
- Hybrid search (semantic + keyword)
\`\`\`

```python
import pytest
from pathlib import Path
import subprocess
import json

@pytest.fixture
def temp_corpus(tmp_path):
    """Create temporary corpus for testing."""
    corpus_path = tmp_path / "test_corpus"
    return corpus_path

def test_end_to_end_embeddings_workflow(temp_corpus):
    """Test complete workflow: ingest → query → verify."""
    # Step 1: Ingest with embeddings
    result = subprocess.run([
        "pdf2llm",
        "--in", "test_pdfs",
        "--out", str(temp_corpus),
        "--parser", "unstructured",
        "--emit-embeddings",
        "--clean"
    ], capture_output=True, text=True)

    assert result.returncode == 0

    # Step 2: Verify vector store files created
    assert (temp_corpus / "_embeddings.faiss").exists()
    assert (temp_corpus / "_chunk_map.json").exists()

    # Step 3: Load chunk map and verify
    chunk_map = json.loads((temp_corpus / "_chunk_map.json").read_text())
    assert "chunk_ids" in chunk_map
    assert len(chunk_map["chunk_ids"]) > 0

    # Step 4: Query corpus
    result = subprocess.run([
        "python", "-m", "pdf2llm.query",
        "--corpus", str(temp_corpus),
        "--query", "plain text",
        "--top-k", "3",
        "--format", "json"
    ], capture_output=True, text=True)

    assert result.returncode == 0

    # Step 5: Verify query results
    query_results = json.loads(result.stdout)
    assert "results" in query_results
    assert len(query_results["results"]) <= 3
    assert all(r["score"] > 0 for r in query_results["results"])
```

class VectorDB(str, Enum):
    """Vector database choices."""
    FAISS = "faiss"
    CHROMA = "chroma"  # Future
    PINECONE = "pinecone"  # Future

vector_db: VectorDB = typer.Option(
    "faiss",
    "--vector-db",
    help="Vector database to use (currently only 'faiss' supported)"
)

# In main():
if config.emit_embeddings and vector_db != "faiss":
    typer.echo(f"Error: --vector-db '{vector_db}' not yet implemented. Use 'faiss'.", err=True)
    raise typer.Exit(code=1)

**Add to README Troubleshooting**:

```markdown
### Known Limitations (Embeddings)

- **CPU only**: GPU acceleration available via faiss-gpu package
- **English optimized**: Model works best for English text
- **Exact search**: Uses brute-force for now; ANN indices for scale in future
- **Full reindex**: Incremental updates not supported; must re-ingest for changes
- **Memory intensive**: Large corpora (>100k chunks) may need significant RAM
```
