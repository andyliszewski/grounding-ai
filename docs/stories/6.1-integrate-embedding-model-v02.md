### Implementation Guidelines

**Determinism Requirement**:
- Same text input must always produce identical embeddings
- Use `normalize_embeddings=True` for consistent L2-normalized vectors
- No random seed or variation in model inference

[Source: Epic 6 AC #4, Architecture v0.2 Design Philosophy - Deterministic Outputs]

**Error Handling**:
- Handle model download failures gracefully
- Provide clear error messages if internet unavailable
- Log model loading progress for user feedback
- Retry download on transient failures (up to 3 attempts)

[Source: docs/architecture.md Section 8 - Error Handling patterns]

**Performance Considerations**:
- Model loading is one-time cost (~2-3 seconds)
- Cache model in memory after first load (singleton pattern)
- Batch encoding not needed for this story (will be added in Story 6.2)

[Source: Epic 6 Performance Targets]

### Model Caching Strategy

**Cache Location**: Use sentence-transformers default cache
- Linux/Mac: `~/.cache/torch/sentence_transformers/`
- Windows: `%USERPROFILE%\.cache\torch\sentence_transformers\`

**First Run Behavior**:
1. Model not in cache → download from HuggingFace
2. Show progress message to user
3. Save to cache directory
4. Load into memory

**Subsequent Runs**:
1. Model found in cache → load directly
2. No download required
3. Fast startup

[Source: sentence-transformers documentation best practices]
