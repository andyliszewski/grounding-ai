## Dev Notes

### Story Context

This is the foundational research story for Epic 8 (Mathematical Formula Extraction). The goal is to select the most suitable tool for extracting mathematical notation from PDF documents. Based on Epic 8 guidance, **pix2tex** is recommended as the primary choice due to ease of integration and reasonable accuracy (80-85%), with **Nougat** as a fallback if higher accuracy is needed.

[Source: docs/epics/epic-8-mathematical-formula-extraction-v02.md]

### Project Background

pdf2llm is a Python CLI tool (Python >=3.10) that converts PDFs to LLM-ready Markdown artifacts. The project follows these principles:
- **Local-only**: No network calls, privacy-focused
- **Determinism**: Same inputs → identical outputs
- **Open-source**: All dependencies must be open-source compatible
- **Minimal dependencies**: Keep the dependency footprint reasonable

[Source: docs/architecture.md, CLAUDE.md]

### Formula Extraction Tool Selection Criteria

The selection criteria prioritize (in order):
1. **Local processing**: Must work entirely offline (no API calls)
2. **Accuracy**: Formula extraction accuracy for common mathematical notation
3. **Python compatibility**: Must integrate with Python >=3.10
4. **LaTeX quality**: Output LaTeX must be compilable and semantically correct
5. **Licensing**: Open-source, commercial-friendly
6. **Installation simplicity**: Reasonable integration effort (but accuracy is more important)

[Source: Epic 8 Technical Architecture]

### pix2tex (Primary Candidate - RECOMMENDED)

**Why recommended**: Python-native, reasonable accuracy, local processing, simpler integration than Nougat.

**Key Characteristics**:
- **Type**: Python package with transformer-based ML model
- **Accuracy**: 80-85% for common mathematical notation
- **Integration**: Direct pip install, Python API
- **Dependencies**: PyTorch (~500MB), transformers, PIL
- **Model Size**: ~100-200MB (downloads on first use)
- **License**: MIT or Apache 2.0 (check during research)
- **Status**: Actively maintained
- **Repository**: https://github.com/lukas-blecher/LaTeX-OCR (pix2tex)

**Integration Approach**:
```python
from pix2tex.cli import LatexOCR

model = LatexOCR()
latex = model(image)  # Returns LaTeX string
```

**Validation Needed**:
- Confirm accuracy claims with test equations
- Verify Python >=3.10 compatibility
- Document installation process
- Test on macOS (primary dev environment)

[Source: Epic 8 Formula Extraction Tool Selection Criteria]

### Nougat (Fallback Candidate)

**Characteristics**:
- **Type**: Meta's research model for scientific document OCR
- **Accuracy**: 90%+ on scientific papers (state-of-art)
- **Integration**: Transformers library, heavier setup
- **Dependencies**: transformers, torch (~1-2GB models)
- **License**: MIT or Apache (likely)
- **Status**: Research-grade, actively developed
- **Repository**: https://github.com/facebookresearch/nougat

**When to use Nougat**:
- If pix2tex accuracy proves insufficient (<75%) during PoC
- If processing entire scientific papers (Nougat handles full-page context better)
- If GPU is available (Nougat benefits significantly from GPU)

**Trade-offs**:
- Higher accuracy but larger models and GPU preference
- More complex integration
- Better for full-document processing vs isolated formulas

[Source: Epic 8 Alternative Tools]

### Test Equations for PoC

**Simple Equations** (must extract correctly):
- Inline: $E=mc^2$
- Inline: $F=ma$
- Display: $$x = \\frac{-b \\pm \\sqrt{b^2-4ac}}{2a}$$

**Moderate Complexity**:
- Integral: $$\\int_0^{\\infty} e^{-x^2}dx = \\frac{\\sqrt{\\pi}}{2}$$
- Summation: $$\\sum_{n=1}^{\\infty} \\frac{1}{n^2} = \\frac{\\pi^2}{6}$$

**Complex Notation** (stretch goal):
- Matrix: $$\\begin{bmatrix} a & b \\\\ c & d \\end{bmatrix}$$
- Piecewise: $$f(x) = \\begin{cases} x^2 & x \\geq 0 \\\\ -x^2 & x < 0 \\end{cases}$$

**Test Data Sources**:
- arXiv papers (public domain scientific papers)
- Mathematics textbooks (public domain via Project Gutenberg)
- Generated LaTeX → PDF → extract → compare

### Success Criteria for PoC

The PoC is successful if:
- pix2tex extracts LaTeX from at least 75% of test equations
- Simple equations (E=mc^2, quadratic formula) extracted with >90% accuracy
- LaTeX output compiles without syntax errors
- Moderate equations extracted with reasonable fidelity (may have minor formatting differences)

If PoC fails:
- Document specific failure modes
- Attempt Nougat PoC as fallback
- Report findings to inform Story 8.2 implementation decisions
