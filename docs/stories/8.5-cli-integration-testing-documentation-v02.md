### Controller Routing Logic

```python
# In controller.py
def run_controller(pdf_path, config):
    # Parse text (always)
    text_elements = parse_pdf(pdf_path, config.ocr_mode)

    if config.extract_formulas:
        # Extract formulas
        formula_elements = extract_formulas(pdf_path)

        # Format formulas
        latex_formulas = formula_formatter.format_to_latex(formula_elements)
        if config.formula_format in ["mathml", "both"]:
            mathml_formulas = formula_formatter.format_to_mathml(formula_elements)

        # Merge with text
        markdown = merge_text_and_formulas(text_elements, formula_elements)

        # Write formula files
        write_formula_files(output_dir, latex_formulas, mathml_formulas)
    else:
        # Existing text-only processing
        markdown = format_to_markdown(text_elements)

    # Continue with chunking, etc.
    ...
```

### Performance Considerations

**Issue Identified: PERF-001 - Model Re-initialization Overhead**

**Finding:**
- pix2tex model is re-initialized for each PDF in batch processing mode
- Adds ~1-2s overhead per file (multiplicative in batch runs)
- No caching mechanism currently implemented

**Impact:**
- Batch processing 10 PDFs: ~10-20s unnecessary overhead
- Single PDF processing: Minimal impact (~2s)
- Scales poorly with file count

**Recommendation:**
Implement lazy singleton pattern for model caching in `formula_extractor.py`:

```python
# Add to formula_extractor.py
_model_instance = None

def get_model():
    """Get or create singleton pix2tex model instance."""
    global _model_instance
    if _model_instance is None:
        from pix2tex.cli import LatexOCR
        _model_instance = LatexOCR()
    return _model_instance

def extract_formulas(pdf_path: Path) -> List[FormulaElement]:
    """Extract formulas using cached model instance."""
    model = get_model()
    # ... rest of extraction logic
```

**Effort**: ~30 minutes
**Impact**: Reduces batch processing overhead by 50-70%
**Priority**: Should fix before production batch workloads

- **Fix**: Changed bracket validation from ERROR (raise) → WARNING (log only)

  - Braces remain strict (critical for LaTeX parsing)
  - Fixes formula_formatter.py lines 230-236
- **Test Updated**: `tests/test_formula_formatter.py::test_validate_latex_syntax_unbalanced_brackets`
- **Status**: ✅ FIXED - Formula extraction no longer crashes on edge cases

**BUG-002: Stats Object Type Error**
- **File**: `pdf2llm/controller.py:525`
- **Symptom**: `AttributeError: 'dict' object has no attribute 'add_chunks'`
- **Root Cause**: Variable shadowing - local variable `stats` overwrote ProcessingStats object
- **Impact**: Controller crashed after formula processing when calling `stats.add_chunks()`
- **Fix**: Renamed local variable from `stats` → `chunk_formula_stats` to avoid shadowing
  - Fixes controller.py lines 525-531
  - Preserves outer `stats` (ProcessingStats) object
- **Status**: ✅ FIXED - Controller completes formula processing successfully
