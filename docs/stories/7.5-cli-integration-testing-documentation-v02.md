## Dev Notes

### CLI Integration Pattern

pdf2llm uses **argparse** for CLI. Current parser flag pattern:

```python
parser.add_argument(
    "--parser",
    choices=["unstructured", "marker"],
    default="unstructured",
    help="PDF parser to use (default: unstructured)",
)
```

**Updated to include OMR support**:
```python
parser.add_argument(
    "--parser",
    choices=["unstructured", "marker", "omr", "hybrid"],
    default="unstructured",
    help="PDF parser: unstructured, marker, omr (music notation), hybrid (auto-detect text/music) (default: unstructured)",
)

parser.add_argument(
    "--music-format",
    choices=["musicxml", "abc", "midi", "all"],
    default="musicxml",
    help="Music output format: musicxml, abc, midi, all (default: musicxml)",
)
```

### README Example Section

```markdown
## OMR Support (Music Notation Extraction)

pdf2llm can extract music notation from PDF sheet music using Audiveris OMR.

### Prerequisites
- Java Runtime Environment (JRE) >=11
- Audiveris >=5.3 (see [installation guide](docs/epics/epic-7-installation-guide.md))

### Process music notation PDFs
```bash
pdf2llm --in ./sheet-music --out ./corpus --parser omr --music-format musicxml
```

### Process hybrid documents (text + music)
```bash
pdf2llm --in ./music-textbooks --out ./corpus --parser hybrid --music-format all
```

### Output formats
- `musicxml`: Industry standard (default, best for analysis)
- `abc`: Lightweight text format (web-friendly)
- `midi`: Playback format
- `all`: Generate all formats

### Query music corpus
```bash
python -m pdf2llm.query --corpus ./corpus --query "I-IV-V progression" --top-k 5
```

See [Epic 7 documentation](docs/epics/epic-7-omr-support-v02.md) for details.
```

### Architecture Diagram Update

Add to architecture.md:

```
OMR Pipeline:
CLI (--parser omr)
 └── Controller
      ├── OMR Parser (Audiveris via py4j)
      ├── Music Formatter (MusicXML/ABC/MIDI)
      ├── Hybrid Processor (for --parser hybrid)
      │    ├── Region Detector
      │    ├── Text Parser (Unstructured/Marker)
      │    └── Music Parser (OMR)
      ├── Writer (music files + chunks)
      └── Manifest (with music metadata)

Hybrid Pipeline:
PDF → Region Detection → [Text Regions → Text Parser] + [Music Regions → OMR Parser]
                      ↓
                [Text Chunks] + [Music Chunks (phrase-based)]
                      ↓
                Combined Output (sequential chunks)
```
