# Story 6.2: Generate Embeddings During Ingestion Pipeline

**Epic:** Epic 6 - Vector Embeddings & Semantic Search
**Status:** Done
**Owner:** Scrum Master (Bob)
**Created:** 2025-10-16
**Dependencies:** Story 6.1 (Integrate Embedding Model)

---

## Story

**As a** PDF processing system,
**I want** to generate vector embeddings for each text chunk during the ingestion pipeline,
**so that** downstream tools can perform semantic search on the corpus.

---

## Acceptance Criteria

1. Embeddings generated for all chunks when `--emit-embeddings` flag is enabled
2. Pipeline performance remains acceptable (<5s per chunk including embedding)
3. Embeddings are accessible during the write phase
4. CLI flag properly toggles embedding generation (off by default)
5. Integration tests verify embeddings are generated for each chunk
6. Embeddings stored in memory during processing, ready for vector store (Story 6.3)

### Embedding Generation Logic

**Pseudocode**:
```python
# In pdf2llm/pipeline.py, after chunking

if config.emit_embeddings:
    logger.info(f"Generating embeddings for {len(chunks)} chunks")

    for chunk in chunks:
        try:
            chunk_text = chunk.text  # Extract text content
            embedding = generate_embedding(chunk_text)
            chunk_id = chunk.metadata['chunk_id']
            config.embeddings[chunk_id] = embedding
        except Exception as e:
            logger.error(f"Failed to generate embedding for chunk {chunk_id}: {e}")
            # Continue with next chunk
```

**Performance Consideration**:
- Sequential processing acceptable for MVP
- Batch processing can be added in future optimization
- Target: <100ms per chunk embedding

[Source: Epic 6 Performance Targets]

### Error Handling Strategy

**Scenarios**:
1. **Model load failure**: Fail fast with clear error message
2. **Single chunk embedding failure**: Log error, continue with next chunk
3. **Empty chunk text**: Skip embedding, log warning
4. **Performance degradation**: Warn if >1s per chunk

**Implementation**:
```python
try:
    embedding = generate_embedding(chunk_text)
except Exception as e:
    logger.error(f"Embedding failed for chunk {chunk_id}: {e}", exc_info=True)
    continue  # Don't fail entire pipeline
```

### Memory Management

**Consideration**: Embeddings stored in memory during processing
- 384 floats × 4 bytes = ~1.5KB per embedding
- 1000 chunks = ~1.5MB
- 10,000 chunks = ~15MB
- Memory overhead is acceptable

**Cleanup**: Embeddings dict cleared after vector store write (Story 6.3)
