## Story Description

As a **developer**, I want to **write each processed document and its chunk files to disk** so that downstream consumers have a deterministic directory structure (`doc.md`, `chunks/ch_####.md`) ready for manifests and QA diffing.

This story introduces the writer module that receives `FileContext` + chunk data from the pipeline (Stories 3.x). It creates slug-based directories, persists the full document Markdown, and writes chunk files with front matter using atomic operations to prevent partial writes.

---

## Context

### From PRD v0.2  
[Source: docs/prd.md — Sections "Output Guarantees" & "Batch Processing"]

- Output layout must match architecture spec (per-document folder, `doc.md`, `chunks/`).  
- Files are expected to be stable across runs (no extra whitespace, consistent naming).  
- Atomic writes protect against corruption when jobs terminate unexpectedly.

### From Architecture v0.2  
[Source: docs/architecture.md — Sections 5 Data Contracts & 6 Output Writer]

- Output path: `<out>/<slug>/doc.md` and `<out>/<slug>/chunks/ch_0001.md`.  
- Writer invoked after chunk metadata assembly (Story 3.3).  
- `FileContext` contains slug, hashes, chunk metadata; writer should not recompute those values.

### From Epic 4  
[Source: docs/epics/epic-4-output-manifest-v02.md — Story 4.1]

- Create slug directory if missing.  
- Write full document Markdown plus chunk files.  
- Use deterministic zero-padded chunk naming.  
- Ensure atomic writes and UTF-8 encoding.

## Acceptance Criteria

### AC1: Writer API
- ✅ Introduce `pdf2llm/writer.py` with `write_document(context: FileContext, markdown: str, chunks: Sequence[str], *, dry_run: bool = False) -> None`.  
- ✅ Function prepares `<output_dir>/<slug>/doc.md` and `<output_dir>/<slug>/chunks/ch_####.md`.  
- ✅ Uses `atomic_write` and `ensure_dir` utilities for safety.

### AC2: Chunk Ordering & Naming
- ✅ Chunk files named `ch_0001.md`, `ch_0002.md`, etc., matching the order supplied.  
- ✅ Empty chunk list still writes `doc.md` and leaves `chunks/` empty.  
- ✅ Overwrites existing files atomically (no partial writes).

### AC3: Observability & Error Handling
- ✅ Logs summary at INFO level (`slug`, `doc_path`, chunk count).  
- ✅ Raises descriptive exception if required paths missing when not in dry-run mode.  
- ✅ `dry_run` support: when enabled, log planned operations without writing files.

---

## Implementation Guidance

1. **Module:** Create `pdf2llm/writer.py` with writer function(s) and module-level logger (`pdf2llm.writer`).  
2. **Inputs:** Accept `FileContext` (from pipeline), raw Markdown, and already-rendered chunk strings.  
3. **Paths:** Use `context.output_path` for doc file; chunk directory is `context.output_path.parent / "chunks"`.  
4. **Atomic Writes:** Utilize existing `atomic_write` helper; ensure parent directories created via `ensure_dir`.  
5. **Determinism:** Do not trim or modify content; rely on upstream normalization.
