### FAISS Technical Details

**Index Type**: Flat L2 (exact search)
- Simple brute-force search
- No compression or approximation
- Suitable for up to 100k vectors
- Exact nearest neighbor results

**Index Properties**:
- Dimension: 384 (matches embedding model)
- Metric: L2 distance (Euclidean)
- Training: Not required for Flat index

**Why Flat L2**:
- Simple and deterministic
- No tuning parameters
- Exact results (important for MVP)
- Fast enough for expected corpus sizes

[Source: Epic 6 Technical Architecture - Vector Store]

### FAISS API Usage

**Index Creation**:
```python
import faiss
import numpy as np

# Create flat L2 index
dimension = 384
index = faiss.IndexFlatL2(dimension)

# Add embeddings (must be float32 matrix)
embeddings_matrix = np.array(list(embeddings.values()), dtype=np.float32)
index.add(embeddings_matrix)

# Save to disk
faiss.write_index(index, str(output_dir / "_embeddings.faiss"))
```

**Index Loading**:
```python
index = faiss.read_index(str(corpus_dir / "_embeddings.faiss"))
```

**Similarity Search**:
```python
# query_embedding: np.ndarray (384,)
# top_k: int

query = query_embedding.astype(np.float32).reshape(1, -1)
distances, indices = index.search(query, top_k)

# indices: array of index positions
# distances: array of L2 distances (lower is better)
```

[Source: FAISS documentation and Epic 6 Technical Architecture]

### Chunk Map Structure

**Format**: JSON file with chunk ID mapping

**Structure**:
```json
{
  "dimension": 384,
  "index_size": 1234,
  "created_utc": "2025-10-16T...",
  "chunk_ids": [
    "141dd45b-0001",
    "141dd45b-0002",
    "9f1b787c-0001",
    ...
  ]
}
```

**Purpose**:
- Map FAISS index position (0, 1, 2, ...) to chunk_id
- `chunk_ids[i]` corresponds to embedding at index position `i`
- Needed because FAISS only returns indices, not IDs

[Source: Epic 6 Technical Architecture and common FAISS patterns]

### Performance Considerations

**Index Build Time**:
- Target: <10 seconds for 10k chunks
- Flat index is fast to build (O(n))
- No training required

**Search Performance**:
- Target: <100ms for top-10 search
- Flat L2 search is O(n) but fast for reasonable corpus sizes
- 100k vectors: ~50ms on modern CPU

**Memory Usage**:
- Index in memory: ~384 floats × 4 bytes × n chunks
- 10k chunks: ~15MB
- 100k chunks: ~150MB

[Source: Epic 6 Performance Targets]

- Same FAISS version
- Same CPU architecture (typically)
- Linux, macOS, Windows
