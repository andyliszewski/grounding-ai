## Story

**As a** user or LLM agent,
**I want** to query a PDF corpus using natural language,
**so that** I can find semantically relevant chunks without exact keyword matching.

3. Results include chunk content and metadata from YAML front matter
4. CLI provides user-friendly formatted output

6. Query module is usable both as CLI and as importable Python module

### Score Normalization

**FAISS Returns**: L2 distances (lower is better)
**User Expects**: Similarity scores (higher is better, 0-1 range)

**Normalization**:
```python
def normalize_scores(distances: np.ndarray) -> np.ndarray:
    """Convert L2 distances to similarity scores.

    Args:
        distances: Array of L2 distances from FAISS

    Returns:
        Similarity scores in range [0, 1]
    """
    # Option 1: Exponential decay
    scores = np.exp(-distances)

    # Option 2: Inverse distance (with offset)
    # scores = 1 / (1 + distances)

    return scores
```

**Note**: Exact normalization function can be tuned based on testing

### Error Handling

**Scenarios**:
1. **Corpus directory doesn't exist**: Clear error with path
2. **No vector index files**: Suggest running with --emit-embeddings
3. **Empty query**: Reject with helpful message
4. **top_k > corpus size**: Adjust to corpus size, warn user
5. **Chunk file missing**: Log warning, skip result

**Error Messages**:
```python
if not corpus_path.exists():
    raise FileNotFoundError(
        f"Corpus directory not found: {corpus_path}\n"
        f"Run 'pdf2llm --in ./pdfs --out {corpus_path} --emit-embeddings' first."
    )

if not (corpus_path / "_embeddings.faiss").exists():
    raise FileNotFoundError(
        f"Vector index not found in {corpus_path}\n"
        f"Re-run ingestion with --emit-embeddings flag to generate embeddings."
    )
```
