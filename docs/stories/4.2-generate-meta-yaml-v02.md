## Story Description

As a **developer**, I want to **emit a per-document `meta.yaml` file** containing provenance, tooling, and hash information so that downstream manifest generation and QA tooling can reference document metadata without re-reading the entire PDF.

The writer (Story 4.1) will call this helper to serialize the metadata alongside `doc.md`. Fields are aligned with the architecture contract, including doc IDs, tool versions, pipeline parameters, and hash fingerprints produced in Epic 3.

---

## Context

### From PRD v0.2  
[Source: docs/prd.md — Sections "Integrity & Auditing" & "Metadata Requirements"]

- Per-document metadata must capture source filename, tooling versions, and hashes.  
- YAML format preferred for readability and compatibility with existing orchestration.  
- Deterministic serialization ensures diffs reflect real content changes.

### From Architecture v0.2  
[Source: docs/architecture.md — Sections 5 Data Contracts & 6 Output Writer]

- `meta.yaml` schema includes: `doc_id`, `slug`, `orig_name`, `tooling`, `params`, `hashes`.  
- Tooling captures parser/formatter/chunker versions.  
- Hashes include file SHA-1 and document content digests.

### From Epic 4  
[Source: docs/epics/epic-4-output-manifest-v02.md — Story 4.2]

- Generate metadata structure and write to YAML.  
- Detect tool versions automatically.  
- Reflect CLI/config parameters used in the run (chunk size, overlap, OCR mode, parser).

## Acceptance Criteria

### AC1: Metadata Builder
- ✅ Implement `build_meta_yaml(context: FileContext, *, params: dict[str, Any], tooling: dict[str, str]) -> str` in `pdf2llm/meta.py` (new module).  
- ✅ Output YAML contains doc ID, slug, original name, tooling versions, parameters, hashes (file SHA-1 + doc hashes).  
- ✅ Deterministic key ordering for stable diffs.

### AC2: Tooling & Params Detection
- ✅ Collect versions for parser, formatter (Marker), chunker (LangChain), hashing (BLAKE3/SHA-256).  
- ✅ Parameters include OCR mode, chunk size/overlap, fallback flags, other pipeline config.  
- ✅ Missing hash data logged as warning but does not crash.

### AC3: Serialization & Testing
- ✅ YAML valid and parseable via `yaml.safe_load`.  
- ✅ Values mirror pipeline state (no recomputation).  
- ✅ Function returns string for writer to persist using atomic write.

---

## Implementation Guidance

1. **Module:** Add `pdf2llm/meta.py` with metadata dataclass/helper functions to prepare YAML-friendly dict.  
2. **Ordering:** Use `OrderedDict` or manual ordered tuples to enforce predictable key order.  
3. **Version Detection:** Reuse existing utilities (e.g., `importlib.metadata.version`) with graceful fallbacks (`unknown`).  
4. **Integration:** Writer (Story 4.1) will call `build_meta_yaml` and write the result via `atomic_write` to `<slug>/meta.yaml`.
