## Story

**As a** music theory student or researcher,
**I want** to semantically search music notation using natural language queries,
**so that** I can find specific harmonic progressions, rhythmic patterns, or musical concepts across a corpus of music PDFs.

---

## Acceptance Criteria

1. Musical content descriptions generated from MusicXML analysis
2. Descriptions include: key signature, time signature, harmonic progression, rhythmic patterns
3. Music21 analysis extracts musical features accurately
4. Embeddings generated using existing Epic 6 model (all-MiniLM-L6-v2)
5. Vector store includes music embeddings alongside text embeddings
6. Query interface returns relevant music phrases for semantic queries
7. `--emit-music-embeddings` CLI flag controls music embedding generation
8. Integration test: Query "I-IV-V progression" returns correct music chunks
9. Performance: Music embedding generation <100ms per phrase

## Dev Notes

### Music Description Generation Strategy

**Goal**: Convert music notation → natural language text → embedding

**Example Music Phrase**:
- MusicXML input: 4 measures in C major, 4/4 time, notes C-D-E-F-G-A-B-C

**Generated Description**:
```
"Music phrase in C major, 4/4 time signature. Ascending scale pattern from C4 to C5 using quarter notes. Harmonic progression: I - V - I (tonic to dominant resolution). No syncopation, regular rhythmic pattern."
```

This description is then embedded using the existing Epic 6 text embedding model.

[Source: Epic 7 Story 7.6 requirements, Epic 7 Decisions Made - Embedding Music Notation]

### music21 Analysis Capabilities

**Key Signature Analysis**:
```python
key_result = stream.analyze('key')
print(key_result.tonic.name, key_result.mode)  # "C major"
```

**Harmonic Analysis (Chord Detection)**:
```python
chords = stream.chordify()  # Reduces to chord stream
roman_numerals = []
for chord in chords.flatten().getElementsByClass('Chord'):
    rn = roman.romanNumeralFromChord(chord, key_result)
    roman_numerals.append(str(rn))
# Example output: ['I', 'IV', 'V', 'I']
```

**Rhythmic Analysis**:
```python
for note in stream.flatten().notes:
    duration = note.quarterLength  # 1.0 = quarter note, 0.5 = eighth, etc.
    # Detect syncopation by checking if note starts on weak beat
```

[Source: music21 documentation, music theory analysis patterns]

### Integration with Epic 6 Embeddings

**Epic 6 Infrastructure** (already implemented):
- `pdf2llm/embedder.py`: generate_embedding(text: str) → np.ndarray (384-dim)
- `pdf2llm/vector_store.py`: write_vector_index(), read_vector_index()
- `pdf2llm/query.py`: query_corpus(query, top_k)
- Model: all-MiniLM-L6-v2 (sentence-transformers)

**Music Embeddings Workflow**:
1. Music phrase (MusicXML) → music21 Stream
2. music21 analysis → musical features (key, harmony, rhythm)
3. Generate text description from features
4. `embedder.generate_embedding(description)` → 384-dim vector
5. Store in same FAISS index as text chunks
6. Query works identically: embed query text, find nearest neighbors

**No new infrastructure needed** - reuse Epic 6 components!

[Source: docs/epics/epic-6-vector-embeddings-v02.md, pdf2llm/embedder.py]

### Description Template

**Template for consistent descriptions**:

```python
def generate_music_description(stream: music21.Stream) -> str:
    key = stream.analyze('key')
    time_sig = stream.getTimeSignatures()[0]
    chords = analyze_harmony(stream)
    rhythm = analyze_rhythm(stream)

    description = f"""
    Music phrase in {key.tonic.name} {key.mode}, {time_sig.ratioString} time signature.
    Harmonic progression: {' - '.join(chords)}.
    Rhythmic pattern: {rhythm}.
    Musical characteristics: {additional_features}.
    """
    return description.strip()
```

**Rationale**: Structured template ensures consistent vocabulary for better embedding quality.

[Source: Best practices for embedding generation]

### Vector Store Schema Extension

**_chunk_map.json entry for music chunk**:
```json
{
  "chunk_id": "abc123_ch_0005",
  "doc_id": "abc123",
  "embedding_index": 4,
  "is_music": true,
  "music_metadata": {
    "phrase_num": 1,
    "key": "C major",
    "time_signature": "4/4",
    "harmony": ["I", "IV", "V", "I"],
    "rhythm": "quarter notes, regular pattern"
  },
  "description": "Music phrase in C major, 4/4 time...",
  "file_path": "beethoven-sonata/chunks/ch_0005.md"
}
```

**FAISS Index**: Music and text embeddings stored together, differentiated by is_music flag.

[Source: pdf2llm/vector_store.py patterns]


Results: Music chunks with syncopation (notes on weak beats).

**Example 3: Key-Based Search**
```bash
python -m pdf2llm.query --corpus ./music-corpus --query "music in C minor" --top-k 5
```

Results: Music chunks in C minor key.

### Performance Considerations

**Target**: <100ms per phrase for embedding generation

**Breakdown**:
- music21 analysis: ~30-50ms (key, harmony, rhythm)
- Description generation: ~5ms (string formatting)
- Embedding generation: ~20-30ms (all-MiniLM-L6-v2 inference)
- **Total**: ~60-85ms (well under 100ms target)

**Optimization if needed**:
