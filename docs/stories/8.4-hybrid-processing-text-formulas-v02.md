## Dev Notes

### Hybrid Document Use Case

**Primary Use Case**: Scientific papers, mathematics textbooks, technical documentation with embedded equations.

**Example Document**:
- Page 1: Introduction text
- Page 1: Display equation defining a variable
- Page 1: Continued text referencing the equation
- Page 2: Derivation with multiple inline equations

**Processing Goal**:
- Extract text as markdown
- Extract formulas as LaTeX
- Embed formulas at correct positions in text: "...as shown in $E=mc^2$, we see..."

[Source: Epic 8 Use Cases]

### Reading Order Preservation

**Challenge**: Formulas must appear in correct position within text flow.

**Strategy**:
1. Extract text with Unstructured/Marker (get text blocks with positions)
2. Extract formulas with formula_extractor (get formulas with bboxes)
3. Sort all elements by (page_num, y_position, x_position)
4. Merge into single stream preserving order

**Example**:
```
Page 1:
  TextBlock (y=100): "Consider the equation"
  Formula (y=150): "E=mc^2"
  TextBlock (y=200): "which relates energy and mass"

Result:
"Consider the equation $$E=mc^2$$ which relates energy and mass"
```

[Source: Epic 8 Story 8.4 requirements]

### Inline vs Display Equation Handling

**Inline equations**: Embedded within paragraphs, use single $ delimiters
- Example: "The value of $\pi \approx 3.14159$ is irrational."

**Display equations**: Centered on own line, use $$ delimiters
- Example:
```markdown
The Pythagorean theorem:
$$a^2 + b^2 = c^2$$
applies to right triangles.
```

**Detection**:
- Use formula_type field from FormulaElement
- Or heuristic: centered formulas (x-position near page center) are display

[Source: LaTeX/Markdown conventions]

### Chunk Metadata Extensions

**Chunk front matter additions**:
```yaml
doc_id: "abc123"
source: "scientific_paper.pdf"
chunk_id: 5
formula_count: 3
inline_formula_count: 2
display_formula_count: 1
formula_ids: ["formula_001", "formula_002", "formula_003"]
hash: "<blake3-of-chunk>"
created_utc: "<ISO8601>"
```

[Source: Epic 8 Formula Metadata Extensions]

### Integration Approach

**Option 1: Extend controller.py** (simpler, for MVP):
```python
def run_controller(pdf_path, config):
    if config.extract_formulas:
        # Parse text
        text_elements = parse_pdf(pdf_path, ocr_mode)

        # Extract formulas
        formula_elements = extract_formulas(pdf_path)

        # Merge and format
        combined_markdown = merge_text_and_formulas(text_elements, formula_elements)

    else:
        # Existing text-only processing
        ...
```

**Option 2: Create hybrid_processor.py** (cleaner separation):
- New module for formula-specific processing
- Keeps controller.py focused on text-only path
- Easier to test formula integration independently

**Recommendation**: Start with Option 1 (extend controller), refactor to Option 2 if complexity grows.

[Source: Architecture best practices]
