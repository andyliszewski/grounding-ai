```python
"""CLI entry point for pdf2llm."""
import typer

app = typer.Typer()

@app.command()
def main(version: bool = typer.Option(False, "--version", help="Show version")):
    """Convert PDFs to LLM-ready Markdown chunks."""
    if version:
        from pdf2llm import __version__
        typer.echo(f"pdf2llm version {__version__}")
        return
    typer.echo("pdf2llm CLI - implementation coming in Story 1.2")

if __name__ == "__main__":
    app()
```

## Technical Notes

### Dependency Rationale

| Dependency | Purpose | Why This Version? |
|------------|---------|-------------------|
| `unstructured>=0.15.0` | PDF parsing with layout preservation | Stable release with OCR support |
| `marker>=0.2.0` | Markdown conversion from parsed PDFs | Proven table/heading preservation |
| `langchain-text-splitters>=0.2.0` | RecursiveCharacterTextSplitter | Industry-standard chunking |
| `typer>=0.9.0` | CLI framework | Clean API, type hints, auto help |
| `tqdm>=4.66.0` | Progress bars | Visual feedback for batch processing |
| `pyyaml>=6.0` | YAML metadata serialization | Metadata front matter generation |
| `ujson>=5.10` | Fast JSON for `_index.json` | Faster than stdlib `json` |
| `blake3>=0.4.1` | Content hashing | Fast, collision-resistant hashes |

### Known Issues / Risks

**Risk:** Unstructured installation may fail on some systems due to native dependencies

**Mitigation:** Document troubleshooting steps in README, provide fallback to Marker-only mode if needed

**Risk:** Dependency version conflicts with user's existing environment

**Mitigation:** Use virtual environment, pin only major versions (e.g., `>=0.15.0` not `==0.15.0`)
