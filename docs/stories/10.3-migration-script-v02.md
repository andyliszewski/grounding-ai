# Story 10.3: Migration Script

**Epic:** Epic 10 - Centralized Corpus Architecture
**Status:** Done
**Owner:** TBD
**Created:** 2025-12-23

---

## Story

**As a** corpus administrator,
**I want** a one-time migration script that merges existing per-agent corpora into a single centralized corpus,
**So that** I can transition from siloed agent corpora to the unified structure.

---

## Acceptance Criteria

1. Migration script discovers all existing agent corpus directories
2. Documents copied to unified corpus without duplication
3. Slug collisions detected and resolved (append doc_id suffix)
4. `source_agent` field populated in each document's meta.yaml
5. Initial `agents/*.yaml` definition files generated
6. Multiple `_index.json` files merged into single manifest
7. Dry-run mode shows what would happen without writing
8. Script provides clear progress output and summary
9. Original directories preserved (copy, not move)

## Dev Notes

### Architecture Context

**Script Location:** `scripts/migrate_corpus.py`

**One-Time Use:** This script runs once during migration. Can be removed after successful migration.

### Data Flow: External → Local → Sync

```
Stage 1: Migration (this script)
┌──────────────────────────────┐         ┌─────────────────────────────┐
│ EXTERNAL DRIVE               │         │ INGESTION MASTER (local)    │
│ My Passport/Agents/          │ ──COPY──► ~/corpus_root/    │
│   ├── CEO/corpus/            │         │   ├── corpus/               │
│   ├── Scientist/corpus/      │         │   ├── originals/ (empty)    │
│   └── Musician/corpus/       │         │   └── agents/               │
└──────────────────────────────┘         └─────────────────────────────┘
                                                      │
Stage 2: Syncthing (ops task, post-migration)        │
                                                      ▼
                                         ┌─────────────────────────────┐
                                         │ CONSUMER MACHINE(s)         │
                                         │ ~/corpus_root/              │
                                         │   ├── corpus/               │
                                         │   └── agents/               │
                                         └─────────────────────────────┘
```

**Critical:** The `--target-dir` must be on **local storage** (not the external drive):
- External drive: Source only, preserved as backup
- Local storage: Where corpus lives after migration
- Syncthing: Distributes from local to consumer machines

**Recommended target location:**
```bash
# Linux ingestion master
--target-dir "~/corpus_root"

# Or a dedicated data directory
--target-dir "/data/corpus_root"
```

**Disk space requirement:** Estimate 2-5GB for processed corpus (depends on source size).

### CLI Usage

```bash

    --source-dir "/Volumes/My Passport/Agents" \
    --target-dir "./corpus_root" \
    --dry-run

# Actual migration
python scripts/migrate_corpus.py \
    --source-dir "/Volumes/My Passport/Agents" \
    --target-dir "./corpus_root"
```

### Collision Resolution

If two agents have documents with the same slug:
- First document: keeps original slug
- Subsequent documents: append `-<doc_id[:8]>` suffix

Example:
```
CEO/corpus/intro-to-business/        → corpus/intro-to-business/
Scientist/corpus/intro-to-business/  → corpus/intro-to-business-7a9b2c1e/
```

### Generated Agent Definition

```yaml
# agents/scientist.yaml
name: scientist
description: "Migrated from Scientist agent corpus"

corpus_filter:
  slugs:
    - ap-biology-for-dummies
    - ap-chemistry-for-dummies
    - physics-fundamentals
```

### Safety Features

1. **Dry-run default:** Script should suggest dry-run first
2. **No deletion:** Original directories preserved
3. **Backup reminder:** Print backup command suggestion
4. **Validation:** Verify merged manifest loads correctly
5. **Idempotent:** Skip documents already in target

### Rollback Procedure

Since migration uses **copy** (not move), rollback is straightforward:

```bash
# If migration fails or needs to be reverted:
rm -rf ./corpus_root/corpus/
rm -rf ./corpus_root/agents/

# Original agent corpora remain intact at:
# /Volumes/My Passport/Agents/*/corpus/

# Re-run migration after fixing issues:
python scripts/migrate_corpus.py --source-dir "..." --target-dir "..." --dry-run
```

Script output should include:
```
Rollback instructions:
  To undo this migration, run:
    rm -rf ./corpus_root/corpus ./corpus_root/agents
  Original data remains at: /Volumes/My Passport/Agents/
```
