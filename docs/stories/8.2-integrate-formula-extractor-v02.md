## Dev Notes

### Architecture Context

pdf2llm follows a modular pipeline architecture. The parser module currently supports Unstructured and Marker parsers using lazy-loading. We'll follow the same pattern for formula extraction.

[Source: docs/architecture.md, pdf2llm/parser.py]

**Lazy Loading Pattern** (from parser.py):
```python
_pix2tex_model = None

def _init_pix2tex_model():
    global _pix2tex_model
    if _pix2tex_model is None:
        from pix2tex.cli import LatexOCR
        _pix2tex_model = LatexOCR()
    return _pix2tex_model
```

[Source: pdf2llm/parser.py:31-44]

### pix2tex Integration Strategy

Based on Story 8.1 research (pix2tex selected), integration follows these steps:

1. **Model Initialization**: Load pix2tex LatexOCR model (lazy load, singleton pattern)
2. **PDF to Images**: Convert PDF pages to images (pdf2image or PyMuPDF)
3. **Formula Detection**: Identify regions likely containing formulas
4. **Extraction**: Pass each formula image to pix2tex model
5. **LaTeX Parsing**: Clean and validate pix2tex output

**Example Usage**:
```python
from pix2tex.cli import LatexOCR

model = LatexOCR()
latex = model(formula_image)  # PIL Image or np.ndarray
print(latex)  # "E=mc^2"
```

[Source: pix2tex documentation, Epic 8 Technical Architecture]

### FormulaElement Data Structure

Represents a single mathematical formula extracted from PDF:

```python
from dataclasses import dataclass
from typing import Optional, Literal, Tuple

@dataclass
class FormulaElement:
    formula_type: Literal["inline", "display"]
    latex_str: str
    page_num: int
    bbox: Tuple[int, int, int, int]  # (x1, y1, x2, y2)

    # Optional fields
    confidence: Optional[float] = None  # 0.0-1.0
    metadata: dict = None  # complexity, domain hints

    def __post_init__(self):
        if self.metadata is None:
            self.metadata = {}
```

This structure will be consumed by `formula_formatter.py` in Story 8.3.

### Formula Detection Approach

**Challenge**: Need to identify which regions of a PDF contain formulas vs regular text.

**Detection Strategy**:

1. **Image-based heuristics** (fast, ~70-80% recall):
   - Convert PDF page to image
   - Find isolated regions with connected component analysis
   - Filter by aspect ratio (formulas wider than tall typically)
   - Filter by position (centered formulas are display equations)

2. **Text-based heuristics** (complement to image):
   - Look for LaTeX delimiters in extracted text ($$, \\[, \\])
   - Dense symbol regions (lots of Greek letters, math operators)

**Implementation**:
```python
def detect_formula_regions(page_image: Image) -> List[BBox]:
    # Convert to grayscale, threshold
    binary = preprocess_image(page_image)

    # Find connected components
    components = find_connected_components(binary)

    # Filter likely formula regions
    formulas = []
    for comp in components:
        if is_likely_formula(comp):
            formulas.append(comp.bbox)

    return formulas
```

### Error Handling Requirements

Following pdf2llm error handling patterns:

**Per-formula error handling**:
- Catch extraction errors for each formula
- Log error with context (page, bbox, error message)
- Continue processing other formulas
- Track failed extractions in statistics

**Specific Errors**:
1. **pix2tex Not Found**: "pix2tex not installed. Install with: pip install pix2tex[full]"
2. **Model Download Failed**: "Failed to download pix2tex model. Check network connection or use --offline mode."
3. **Extraction Failure**: "Failed to extract formula at page X, bbox (x1,y1,x2,y2): [error]. LaTeX may be incomplete."
4. **Invalid LaTeX**: "pix2tex produced invalid LaTeX: [latex]. Skipping this formula."

[Source: pdf2llm error handling patterns, CLAUDE.md]
