**Epic:** Epic 10 - Centralized Corpus Architecture
**Status:** Done
**Owner:** TBD
**Created:** 2025-12-23

---

## Story

**As a** corpus administrator,
**I want** to specify collection tags when ingesting documents via the CLI,
**So that** documents are automatically tagged for agent filtering.

---

## Acceptance Criteria

1. New `--collections` CLI flag accepts comma-separated collection names
2. Collections passed through pipeline to meta.yaml generation
3. Collections appear in generated `meta.yaml`
4. Collections appear in `_index.json` manifest entries
5. Collection names validated (lowercase kebab-case)
6. Invalid collection names rejected with clear error message
7. Help text documents the flag with examples
8. Existing workflows without `--collections` continue to work

### Error Message Examples

```bash
# Invalid collection name (uppercase)
$ pdf2llm --in ./pdfs --out ./corpus --collections Science
Error: Invalid collection name 'Science'. Collections must be lowercase kebab-case (e.g., 'science', 'music-theory').

# Invalid collection name (underscore)
$ pdf2llm --in ./pdfs --out ./corpus --collections music_theory
Error: Invalid collection name 'music_theory'. Collections must be lowercase kebab-case (e.g., 'science', 'music-theory').
```

### CLI Implementation Pattern

```python
# In cli.py
@app.command()
def main(
    # ... existing parameters ...
    collections: Annotated[
        Optional[str],
        typer.Option(
            "--collections",
            help="Comma-separated collection tags for documents (e.g., 'science,biology')"
        )
    ] = None,
):
    # Parse and validate collections
    collection_list = None
    if collections:
        collection_list = [c.strip() for c in collections.split(",")]
        for name in collection_list:
            if not validate_collection_name(name):
                raise typer.BadParameter(
                    f"Invalid collection name '{name}'. "
                    "Collections must be lowercase kebab-case (e.g., 'science', 'music-theory')."
                )

    # Pass to pipeline
    config = PipelineConfig(
        # ... existing config ...
        collections=collection_list,
    )
```

def test_full_pipeline_with_collections(tmp_path, sample_pdf):
    """Test complete pipeline with collections flag."""
    result = runner.invoke(app, [
        "--in", str(sample_pdf.parent),
        "--out", str(tmp_path / "corpus"),
        "--collections", "science,biology"
    ])
    assert result.exit_code == 0

    # Verify meta.yaml
    meta_path = tmp_path / "corpus" / "sample" / "meta.yaml"
    meta = yaml.safe_load(meta_path.read_text())
    assert meta["collections"] == ["science", "biology"]

    # Verify _index.json
    manifest_path = tmp_path / "corpus" / "_index.json"
    manifest = json.loads(manifest_path.read_text())
    assert manifest["docs"][0]["collections"] == ["science", "biology"]
