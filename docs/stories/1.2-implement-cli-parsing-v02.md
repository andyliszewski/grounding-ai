## Story Description

As a **developer**, I need a **CLI interface** that parses user-supplied arguments (input/output directories, chunk configuration, dry-run setting) so that the pipeline can be invoked from the command line consistently.

CLI parsing ensures downstream stories can rely on structured configuration objects. This story focuses on defining CLI flags and mapping them to pipeline configuration without invoking actual processing yet (integration with pipeline occurs in later stories).

---

## Context

### From PRD v0.2  
[Source: docs/prd.md — Sections "Batch Processing" & "User Experience"]

- Users interact via CLI to specify input directory, output directory, and chunk parameters.  
- Dry-run support is required for verification without writing files.  
- CLI should output deterministic logs and help text.

### From Architecture v0.2  
[Source: docs/architecture.md — CLI Appendix]

- CLI flags map to `PipelineConfig`.  
- CLI uses Typer for ergonomics.  
- Future stories integrate CLI with pipeline execution.

---

## Acceptance Criteria

### AC1: CLI Arguments
- ✅ `pdf2llm.cli` accepts `--input/-i`, `--output/-o`, `--chunk-size`, `--chunk-overlap`, `--ocr-mode`, `--dry-run`, `--allow-plaintext-fallback`.  
- ✅ Validates that input directory exists and output directory provided.  
- ✅ Returns a structured configuration object or dataclass.

### AC2: Defaults & Help Text
- ✅ Sensible defaults (chunk size 1200, overlap 150, ocr-mode auto).  
- ✅ Help text references architecture defaults.  
- ✅ `--version` displays project version.

### AC3: Error Handling
- ✅ Invalid paths raise Typer exit with readable message.  
- ✅ Invalid chunk size/overlap produce descriptive errors.  
- ✅ `OCR` mode validated against allowed values.

---

## Implementation Guidance

1. **Typer Setup:** Build CLI using Typer commands with explicit type hints.  
2. **Validation:** Use `Path` validations (`exists`, etc.).  
3. **Output:** Return configuration object; actual execution deferred.  
4. **Logging:** Provide basic logging for preview output.
