## Story Description

As a **developer**, I want to **create integration tests using sample PDFs** so that the end-to-end pipeline is validated for determinism, error handling, and CLI flags.

This story introduces fixtures in `tests/test_integration.py` that exercise the full CLI/controller, verifying output artifacts, manifest entries, and rerun stability.

---

## Context

### From PRD v0.2  
[Source: docs/prd.md — Sections "Quality Assurance" & "Determinism"]

- Must validate full pipeline on sample PDFs, including tables, text-heavy, and malformed files.  
- Determinism requirement: repeated runs produce identical outputs/hashes.  
- Error cases should demonstrate graceful handling.

### From Epic 5
- Integration suite intended to follow Stories 5.1–5.3; ensures readiness for production use.  
- Provides regression coverage for future changes.

---

## Acceptance Criteria

### AC1: Sample Fixtures
- ✅ Add three sample PDFs to `test_pdfs/` (structured table, plain text, malformed).  
- ✅ Provide helper to copy fixtures into temp directories during tests.

### AC2: CLI Integration Test
- ✅ Use Typer `CliRunner` to run CLI command on fixtures.  
- ✅ Assert doc/chunks/meta/manifest files produced with expected counts.  
- ✅ Rerun command and assert outputs unchanged (hash comparison).

### AC3: Error & Flag Coverage
- ✅ Simulate parser failure (e.g., malformed PDF) and assert CLI exits with failure summary while processing remaining files.  
- ✅ Validate `--dry-run`, `--clean`, `--allow-plaintext-fallback`, `--parser` toggles in integration context.  
- ✅ Confirm manifest entries updated correctly after second run.

---

## Implementation Guidance

1. **Fixtures:** Store minimal-size PDFs in `test_pdfs/` (supported by repo).  
2. **Helpers:** Create `tests/integration_utils.py` with functions to copy fixtures and compute hashes.  
3. **Assertions:** Compare output directories to golden directories or compute SHA-1 for `doc.md`/manifest.  
4. **Timeout:** Keep tests performant (<2s) by using small PDFs.
