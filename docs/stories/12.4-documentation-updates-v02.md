### how-to-local-agent.md Updates

Add the following sections after "Usage":

```markdown
## Agentic Mode

Agentic mode enables the LLM to decide when to search your corpus, rather than
always searching before responding. This allows for more natural conversations
where the model can use its own knowledge for simple questions and search only
when needed.

### Simple RAG vs Agentic Mode

| Aspect | Simple RAG | Agentic Mode |
|--------|------------|--------------|
| Search behavior | Always searches first | LLM decides when to search |
| Multi-turn | No | Yes (can search multiple times) |
| Latency | One LLM call | Multiple LLM calls possible |
| Token usage | Fixed | Grows with iterations |
| Best for | Fact-heavy queries | Complex reasoning |

### Enabling Agentic Mode

Add the `--agentic` flag (or `-A`) to your command:

```bash
# Single query
./venv/bin/python scripts/local_rag.py --agent scientist --agentic \
  --query "What's the relationship between entropy and information?"

# Interactive REPL
./venv/bin/python scripts/local_rag.py --agent scientist --agentic
```

### Agentic Mode Options

| Flag | Description | Default |
|------|-------------|---------|
| `--agentic`, `-A` | Enable agentic mode | false |
| `--max-iterations` | Max tool-calling rounds | 5 |
| `--verbose`, `-v` | Show tool calls | false |

### Example: Single Tool Call

```
$ ./venv/bin/python scripts/local_rag.py --agent survivor --agentic -v \
  --query "How do I purify water in the wilderness?"

[Agentic] Iterations: 2
[Agentic] Stopped: complete
[Agentic] Tool calls made: 1
  - search_corpus({'query': 'water purification wilderness survival', 'top_k': 5})

Based on the survival guides in the corpus, here are the main methods
for water purification in the wilderness:

1. **Boiling** - The most reliable method. Bring water to a rolling boil
   for at least 1 minute (3 minutes at high altitude).

2. **Solar disinfection (SODIS)** - Fill clear plastic bottles and leave
   in direct sunlight for 6+ hours...

[Sources: wilderness-survival-guide.pdf, camping-handbook.pdf]
```

### Example: Multi-Turn Tool Calling

```
$ ./venv/bin/python scripts/local_rag.py --agent scientist --agentic -v \
  --query "Compare the thermodynamic efficiency of Carnot and Stirling engines"

[Agentic] Iterations: 3
[Agentic] Stopped: complete
[Agentic] Tool calls made: 2
  - search_corpus({'query': 'Carnot engine efficiency thermodynamics', 'top_k': 5})
  - search_corpus({'query': 'Stirling engine efficiency comparison', 'top_k': 5})

Based on the physics texts in the corpus:

**Carnot Engine:**
The Carnot cycle represents the theoretical maximum efficiency for any heat
engine operating between two temperatures...

**Stirling Engine:**
The Stirling engine, when operating on the ideal Stirling cycle, can achieve
Carnot efficiency because...
```

### Example: LLM Answers Without Searching

Not every query requires a corpus search. Simple questions may be answered
directly:

```
$ ./venv/bin/python scripts/local_rag.py --agent scientist --agentic -v \
  --query "What is 2 + 2?"

[Agentic] Iterations: 1
[Agentic] Stopped: complete
[Agentic] Tool calls made: 0

The answer is 4.
```

### When to Use Each Mode

**Use Simple RAG when:**
- You know you need corpus information
- Query is straightforward fact retrieval
- You want consistent, predictable behavior
- Latency is critical

**Use Agentic Mode when:**
- Query may require multiple searches
- LLM needs to reason about what to search
- Complex multi-part questions
- Conversational interaction style preferred
```

### Troubleshooting Section Addition

```markdown
### Agentic Mode Issues

#### "LLM never calls tools"

Some models may not reliably use tools. Try:
- Use Qwen 2.5 (14B or 32B) - specifically trained for tool use
- Make your question more specific ("search for..." vs "do you know...")
- Check that the tool schema is being sent (use verbose mode)

#### "LLM keeps calling tools forever"

The `--max-iterations` flag prevents infinite loops:

```bash
# Limit to 3 iterations
./venv/bin/python scripts/local_rag.py --agent ceo --agentic \
  --max-iterations 3 --query "..."
```

If hitting the limit often, your corpus may lack relevant information.

#### "Agentic mode is slow"

Each tool call requires a round-trip to Ollama. Expected latency:
- Simple RAG: 1 LLM call (~2-5 seconds)
- Agentic (1 tool call): 2 LLM calls (~4-10 seconds)
- Agentic (3 tool calls): 4 LLM calls (~8-20 seconds)

Consider simple RAG for time-sensitive queries.

#### "Tool returns error messages"

Check your FAISS index:
```bash
# Verify embeddings exist
ls ~/Documents/Corpora/embeddings/<agent>/

# Regenerate if needed
./venv/bin/pdf2llm embeddings --agent <agent> --corpus ~/Documents/Corpora/corpus
```
```

### CLAUDE.md Updates

Add to the Architecture section:

```markdown
### Agentic Tool Calling (Epic 12)

The agentic system enables local LLMs to use tools for corpus search:

**Module Structure:**
- `scripts/agentic.py` - Agent loop and tool calling infrastructure
- `scripts/search_corpus_tool.py` - search_corpus tool implementation

**Key Components:**
- `AgenticConfig`: Configuration (max_iterations, verbose, timeout)
- `ToolRegistry`: Registers tools with schemas and executors
- `run_agentic_loop()`: Main loop handling tool call cycles
- `SearchCorpusTool`: FAISS search wrapped as a tool

**Flow:**
1. User query sent to LLM with tool schemas
2. LLM responds with optional tool_calls
3. If tool_calls present, execute and add results to messages
4. Loop until no tool_calls or max_iterations reached
5. Return final response

**API Compatibility:**
Uses Ollama's OpenAI-compatible `/api/chat` endpoint with `tools` parameter.
```
