## Story Description

As a **developer**, I want to **prepare and manage the output directory** (including `--clean` and dry-run behaviors) so that the pipeline writes files safely without leaving stale artifacts.

This story adds CLI/pipeline plumbing to honor configuration flags, create required directories, validate write permissions, and ensure the manifest is initialized when missing.

---

## Context

### From PRD v0.2  
[Source: docs/prd.md — Sections "Batch Processing" & "Reliability"]

- Users may rerun pipelines and expect `--clean` to remove previous outputs before writing new ones.  
- Dry-run mode should report intended actions without modifying disk.  
- Missing permissions should fail fast with actionable errors.

### From Architecture v0.2  
[Source: docs/architecture.md — Sections 6 Output Writer & CLI Appendix]

- Output directory root contains `_index.json` and per-slug folders.  
- Writer assumes root exists; pipeline must ensure creation.  
- Manifest should be initialized (empty structure) on first run.

### From Epic 4  
[Source: docs/epics/epic-4-output-manifest-v02.md — Story 4.4]

- Implement `--clean` flag to remove existing output.  
- Create directories with correct permissions.  
- Dry-run mode should outline actions.  
- Initialize empty manifest when none exists.

---

## Acceptance Criteria

### AC1: Directory Preparation
- ✅ Extend `PipelineConfig`/CLI to honor `--clean`; when true (and not dry-run), remove output dir before processing.  
- ✅ Ensure output root created with `ensure_dir`.  
- ✅ Validate write permissions and raise descriptive error if not writable.

### AC2: Dry-Run Behavior
- ✅ When `dry_run=True`, log planned directory/manifest actions without deleting or writing files.  
- ✅ Writer/manifest helpers invoked in dry-run should skip disk operations (already supported).  
- ✅ Summary log indicates no files written due to dry run.

### AC3: Manifest Bootstrapping
- ✅ If `_index.json` missing, create empty manifest structure via `ManifestManager` (Story 4.3) before document loop.  
- ✅ After processing, manifest updated and written atomically.  
- ✅ Clean mode removes manifest and per-slug folders prior to run.
---

## Implementation Guidance

1. **CLI Config:** Update CLI parsing to expose `--clean/--no-clean` and dry-run hook if not already present.  
2. **Pipeline Hook:** Before `scan_pdfs`, call setup helper to clean/prepare directories and load manifest.  
3. **State Flow:** Maintain manifest object across run; at completion, write using manager (Story 4.3).  
4. **Logging:** Provide concise info logs for operations (clean, create, dry-run skip).
